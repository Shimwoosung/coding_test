# 그래프 (Graph)

## 한 줄 요약
그래프는 **정점(Vertex/Node)** 들과 그 사이를 잇는 **간선(Edge)** 으로 이루어진 자료구조로, "무엇이 무엇과 연결되어 있는가"라는 **관계**를 표현하는 가장 일반적인 모델이다.

---

## 언제 쓰나
실제 코딩테스트에서 아래 같은 신호가 보이면 거의 그래프 문제다.

- **"도시 A에서 도시 B로 가는 길"**, 도로/항로/네트워크 연결 → 정점=도시, 간선=길
- **미로 / 격자(grid) 탐색** — 상하좌우로 이동, 최단 거리 구하기 (격자도 그래프다! 칸=정점, 인접 칸=간선)
- **친구 관계, 팔로우 관계** — SNS에서 "몇 다리 건너 아는 사이인가"
- **선수과목 / 작업 순서** — "A를 들어야 B를 들을 수 있다" → 위상정렬(Topological Sort)
- **연결 요소 개수 세기** — "섬이 몇 개인가", "그룹이 몇 개로 나뉘는가"
- **사이클 탐지** — "순환 참조가 있는가"
- **최단 경로** — 가중치가 있으면 다익스트라/벨만-포드/플로이드-워셜
- **최소 신장 트리(MST)** — "모든 도시를 잇는 최소 비용 도로망" → 크루스칼/프림

핵심: **"개체들 사이의 연결 관계"** 가 문제의 본질이면 그래프를 의심하라.

---

## 왜 이걸 쓰나 + 대안과 차이

### 다른 자료구조와의 관계
| 자료구조 | 한 줄 정의 | 그래프와의 관계 |
|---|---|---|
| **배열/리스트** | 일렬로 나열 | 그래프의 정점/간선을 저장하는 그릇으로 쓰임 |
| **트리(Tree)** | 사이클 없는 연결 그래프 | 트리는 **그래프의 특수한 경우** (정점 N개, 간선 N-1개, 사이클 없음) |
| **그래프(Graph)** | 자유로운 연결 | 사이클 O, 여러 컴포넌트 O, 가장 일반적 |

> 트리는 그래프의 부분집합이다. 그래서 트리 순회(DFS/BFS) 기법이 그래프에서도 그대로 통한다. 단, 그래프는 **사이클이 있을 수 있어서 "방문했는가(visited)" 체크가 필수**다.

### 그래프 표현 방법 — 인접 행렬 vs 인접 리스트
정점 개수 V, 간선 개수 E라고 하자.

| 표현 | 메모리 | "u와 v가 연결?" 확인 | u의 이웃 전체 순회 | 언제 |
|---|---|---|---|---|
| **인접 행렬** `adj[u][v]` | O(V²) | O(1) 빠름 | O(V) | V가 작고(≤ ~2000), 간선이 빽빽할 때(dense) |
| **인접 리스트** `vector<int> adj[V]` | O(V+E) | O(deg) 느림 | O(deg) 빠름 | V가 크고, 간선이 희박할 때(sparse) — **대부분의 문제** |

코딩테스트에서는 **인접 리스트가 기본**이라고 생각하면 된다. (메모리·속도 모두 유리)

### 탐색 알고리즘 비교
- **DFS (깊이 우선)**: 한 방향으로 끝까지 파고든 뒤 되돌아옴. 재귀/스택. 경로·사이클·연결요소·위상정렬에 강함.
- **BFS (너비 우선)**: 가까운 곳부터 물결처럼 퍼짐. 큐. **가중치 없는 그래프의 최단 거리**에 최적.
- 가중치가 있으면 → **다익스트라(음수 없음)**, **벨만-포드(음수 가능)**, **플로이드-워셜(모든 쌍)**.

## 동작 과정

### 0) 그래프 한 장으로 보기 (무방향, 정점 0~5)
```
        (0)
       /   \
     (1)   (2)
      |    /  \
     (3) (4)  (5)

간선 목록: 0-1, 0-2, 1-3, 2-4, 2-5

인접 리스트 표현:
  0 -> [1, 2]
  1 -> [0, 3]
  2 -> [0, 4, 5]
  3 -> [1]
  4 -> [2]
  5 -> [2]
```

### 1) BFS — 0번에서 시작, 큐가 어떻게 변하는가
가까운 정점부터 "층(level)" 단위로 퍼져나간다. 큐(FIFO: 먼저 들어간 게 먼저 나옴)를 사용.

```
시작: 0을 큐에 넣고 visited 표시

step  큐(front ... back)   꺼낸 정점   새로 넣는 이웃        visited 집합
----  -------------------  ---------  -------------------  ---------------------
 0    [0]                   -          -                    {0}
 1    [1, 2]                0          1,2 push             {0,1,2}
 2    [2, 3]                1          3 push (0은 방문됨)   {0,1,2,3}
 3    [3, 4, 5]             2          4,5 push             {0,1,2,3,4,5}
 4    [4, 5]                3          (이웃 1 방문됨)       (변화 없음)
 5    [5]                   4          (이웃 2 방문됨)       (변화 없음)
 6    []                    5          (이웃 2 방문됨)       (변화 없음)

방문 순서: 0 -> 1 -> 2 -> 3 -> 4 -> 5
거리(0 기준): dist[0]=0, dist[1]=1, dist[2]=1, dist[3]=2, dist[4]=2, dist[5]=2
```
물결 그림으로 보면:
```
   거리0      거리1          거리2
   (0)  -->  (1)(2)  -->  (3)(4)(5)
```

### 2) DFS — 0번에서 시작, 끝까지 파고든다
한 갈래를 끝까지 따라간 뒤, 막히면 직전 갈림길로 **되돌아온다(backtrack)**.

```
방문 흐름 (재귀 호출 스택 깊이로 표현):

visit 0
  -> visit 1            (0의 첫 이웃)
       -> visit 3       (1의 이웃, 막다른 길)
       <- back to 1
  <- back to 0
  -> visit 2            (0의 다음 이웃)
       -> visit 4       (막다른 길)
       <- back to 2
       -> visit 5       (막다른 길)
       <- back to 2
  <- back to 0

방문 순서: 0 -> 1 -> 3 -> 2 -> 4 -> 5
```
트리처럼 그리면 (탐색이 만든 길 = DFS 트리):
```
        0
       / \
      1   2
      |  / \
      3 4   5
   (숫자 옆 화살표 순서로 깊이 우선 진입)
```

### 3) 격자(grid)도 그래프다 — 미로 BFS
`1`=벽, `0`=길. (0,0)에서 (행,열) 상하좌우 이동, 최단 칸 수를 구한다고 하자.
```
  맵            거리 채우기 (BFS가 퍼지는 모습)
  0 0 1         0 1 .          0 1 X
  0 1 0   -->   1 X .   -->    1 X 5
  0 0 0         2 . .          2 3 4

(. 은 아직 미방문, X 는 벽)
시작 (0,0)부터 한 칸씩 거리+1 하며 사방으로 퍼진다.
도착 (2,2)까지 최단 거리 = 4
```

## C++ 예제 코드

```cpp
#include <bits/stdc++.h>
using namespace std;

const int MAXV = 100000;        // 최대 정점 수 (문제에 맞게 조정)
vector<int> adj[MAXV];          // 인접 리스트: adj[u] = u와 연결된 정점들
bool visited[MAXV];             // 방문 체크 (사이클 있어도 무한루프 방지)

// ---------- 1) BFS: 시작점 start에서 각 정점까지의 최단 거리 ----------
// 가중치 없는 그래프에서 "최단 경로"는 BFS가 정답.
vector<int> bfs(int start, int V) {
    vector<int> dist(V, -1);    // -1 = 아직 도달 못함
    queue<int> q;

    dist[start] = 0;            // 시작점 거리 0
    q.push(start);              // 큐에 시작점 투입

    while (!q.empty()) {
        int cur = q.front();    // FIFO: 가장 먼저 들어온 정점 꺼냄
        q.pop();

        for (int next : adj[cur]) {     // cur의 모든 이웃 확인
            if (dist[next] == -1) {     // 아직 방문 안 한 이웃만
                dist[next] = dist[cur] + 1;  // 한 칸 더 멀다
                q.push(next);            // 큐에 넣어 나중에 펼침
            }
        }
    }
    return dist;
}

// ---------- 2) DFS (재귀 버전): start와 연결된 정점 전부 방문 ----------
void dfs(int cur) {
    visited[cur] = true;        // 먼저 방문 표시 (재방문 방지의 핵심)
    cout << cur << ' ';         // 방문 순서 출력

    for (int next : adj[cur]) {
        if (!visited[next]) {   // 아직 안 갔으면
            dfs(next);          // 거기로 더 깊이 파고든다
        }
    }
    // 모든 이웃을 다 봤으면 자연스럽게 호출이 끝나며 backtrack
}

// ---------- 3) DFS (명시적 스택 버전): 재귀 깊이가 깊어 스택오버플로 우려 시 ----------
void dfs_iterative(int start) {
    stack<int> st;
    st.push(start);
    while (!st.empty()) {
        int cur = st.top();
        st.pop();
        if (visited[cur]) continue;     // 스택엔 중복이 들어갈 수 있으므로 여기서 거른다
        visited[cur] = true;
        cout << cur << ' ';
        for (int next : adj[cur])
            if (!visited[next])
                st.push(next);
    }
}

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    int V, E;                   // 정점 수, 간선 수
    cin >> V >> E;

    for (int i = 0; i < E; i++) {
        int u, v;
        cin >> u >> v;
        adj[u].push_back(v);    // u -> v
        adj[v].push_back(u);    // 무방향이면 양쪽 다 추가. (방향 그래프면 이 줄 삭제)
    }

    // BFS로 0번에서의 최단 거리 출력
    vector<int> dist = bfs(0, V);
    for (int i = 0; i < V; i++)
        cout << "dist[" << i << "] = " << dist[i] << '\n';

    // DFS 방문 순서 출력
    fill(visited, visited + V, false);
    cout << "DFS order: ";
    dfs(0);
    cout << '\n';

    return 0;
}
```

> 위 코드를 미로(격자)에 적용하려면 `adj` 대신 `int dx[4]={1,-1,0,0}, dy[4]={0,0,1,-1};` 로 상하좌우 이웃을 계산하고, 경계(범위)·벽 검사를 추가하면 된다. 본질은 같다.

## 시간복잡도
정점 V개, 간선 E개 기준.

| 알고리즘 | 인접 리스트 | 인접 행렬 |
|---|---|---|
| **DFS** | O(V + E) | O(V²) |
| **BFS** | O(V + E) | O(V²) |
| 다익스트라(우선순위 큐) | O((V+E) log V) | — |
| 플로이드-워셜(모든 쌍 최단경로) | — | O(V³) |

- **DFS/BFS 모두 O(V+E)**: 각 정점을 한 번씩 방문(V), 각 간선을 한 번(방향) 또는 두 번(무방향) 훑기(E) 때문.
- 인접 **행렬**을 쓰면 한 정점의 이웃을 찾을 때마다 V칸을 다 훑어야 해서 O(V²)로 늘어난다. → 정점이 많으면 리스트를 써라.
- 공간복잡도: 리스트 O(V+E), 행렬 O(V²).

## 흔한 실수 / 팁

1. **visited 표시 타이밍**
   - BFS는 **큐에 넣는 순간** visited(또는 dist 갱신) 처리해야 한다. 꺼낼 때 표시하면 같은 정점이 큐에 여러 번 들어가 **중복/시간초과** 발생.
   - DFS는 함수에 들어가자마자 표시.

2. **무방향 그래프는 간선을 양쪽에 추가** (`adj[u].push_back(v); adj[v].push_back(u);`). 한쪽만 넣으면 절반의 길이 사라진다.

3. **재귀 DFS 스택오버플로**: 정점이 수십만 개이고 일자로 길게 이어진 경우(예: 0-1-2-...-100000) 재귀 깊이가 터진다. → **반복문(스택) DFS** 또는 BFS로 전환.

4. **연결 요소가 여러 개**일 수 있다. "전체 정점을 모두 처리"해야 하면 **모든 정점에 대해 루프 돌며** 미방문이면 DFS/BFS 시작:
   ```cpp
   int components = 0;
   for (int i = 0; i < V; i++)
       if (!visited[i]) { dfs(i); components++; }   // 섬의 개수 = components
   ```

5. **정점 번호가 0-base인지 1-base인지** 문제마다 다르다. 배열 크기와 루프 범위를 거기에 맞춰라. (보통 PS는 1-base가 많다 → 배열을 V+1로)

6. **가중치 있는 최단경로에 BFS 쓰지 말 것.** BFS는 "간선 1개 = 거리 1"일 때만 최단을 보장한다. 가중치가 있으면 다익스트라.

7. **격자 문제에서 범위 검사 먼저, 그 다음 벽/방문 검사.** `if (nx < 0 || nx >= N || ny < 0 || ny >= M) continue;` 를 빼먹으면 배열 밖 접근으로 런타임 에러.

8. **DFS/BFS 방문 순서는 인접 리스트에 넣은 순서에 따라 달라진다.** "유일한 정답 순서"가 필요한 문제면 이웃을 정렬(`sort(adj[u].begin(), adj[u].end())`)해 두자.
