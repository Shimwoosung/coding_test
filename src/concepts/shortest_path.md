# 최단 경로 (Shortest Path)

## 한 줄 요약
그래프에서 한 정점(또는 모든 정점)에서 다른 정점까지 가는 **간선 가중치의 합이 가장 작은 경로**를 찾는 알고리즘 묶음이며, 그래프의 성격(가중치 유무, 음수 간선 유무, 출발점 개수)에 따라 BFS / 다익스트라(Dijkstra) / 벨만-포드(Bellman-Ford) / 플로이드-워셜(Floyd-Warshall)을 골라 쓴다.

## 언제 쓰나
실제 코딩테스트에서 다음과 같은 문장이 보이면 "최단 경로" 문제일 가능성이 높다.

- "도시 1에서 도시 N까지 가는 **최소 비용**을 구하라" (가중치 그래프 → 다익스트라)
- "미로에서 출발점부터 도착점까지 **최소 칸 수(이동 횟수)**" (가중치가 전부 1 → BFS)
- "모든 도시 쌍 사이의 최단 거리를 출력하라" (전체 쌍 → 플로이드-워셜)
- "환율 차익, 시간 되돌리기 등 **음수 가중치**가 등장하고 음수 사이클을 탐지해야 한다" (벨만-포드)
- "택배 배달 최소 시간", "지하철 환승 최소 요금", "네트워크 패킷 최소 지연" 같은 비용 최소화 문제
- "특정 정점을 **반드시 거쳐 가는** 최단 경로" (출발→경유, 경유→도착 두 번 계산)

## 왜 이걸 쓰나 + 대안과 차이
최단 경로 문제는 "어떤 알고리즘이 정답"이 아니라 **그래프 조건에 따라 최적 도구가 다르다.** 아래 표로 한눈에 비교한다.

| 알고리즘 | 가중치 | 음수 간선 | 출발점 | 시간복잡도 | 핵심 아이디어 |
|----------|--------|-----------|--------|------------|----------------|
| **BFS** | 없음(=모두 1) | 불가 | 1개 | O(V+E) | 큐로 가까운 곳부터 층별 탐색 |
| **다익스트라** | 양수만 | **불가** | 1개 | O(E log V) | 우선순위 큐로 최솟값부터 확정 |
| **벨만-포드** | 임의 | 가능 | 1개 | O(V·E) | 모든 간선을 V-1번 완화(relax) |
| **플로이드-워셜** | 임의 | 가능 | 전체 쌍 | O(V³) | 경유지 k를 하나씩 늘려가며 갱신 |

**왜 BFS로는 가중치 그래프를 못 푸나?**
BFS는 "간선 1개 = 거리 1"을 전제로 큐에서 먼저 꺼낸 정점이 가장 가깝다고 믿는다. 가중치가 다르면 이 전제가 깨진다. 간선이 3인 길을 한 번에 가는 것보다, 간선이 1인 길을 두 번 가는 게 더 짧을 수 있기 때문이다.

**왜 다익스트라는 음수 간선에서 틀리나?**
다익스트라는 "한 번 최단 거리로 확정한 정점은 다시 줄어들지 않는다"는 **탐욕적(greedy)** 가정으로 동작한다. 음수 간선이 있으면 이미 확정한 정점이 나중에 더 짧아질 수 있어서 이 가정이 무너진다. 이럴 땐 벨만-포드를 쓴다.

**왜 전체 쌍이면 플로이드-워셜인가?**
다익스트라를 정점마다 V번 돌리면 O(V·E log V)이다. 그래프가 조밀(E ≈ V²)하고 V가 작으면(보통 V ≤ 400~500) 구현이 3중 for문으로 끝나는 플로이드-워셜이 더 간단하고 충분히 빠르다.

## 동작 과정

### (1) BFS — 가중치 없는 격자 미로 (모든 이동 비용 = 1)
`.`은 길, `#`은 벽, `S`는 출발, `E`는 도착. 출발점부터 거리(층)를 차례로 채운다.

```
[격자]            [BFS 거리 전파: 큐에서 가까운 칸부터]
S . . #           0 1 2 #
. # . .           1 # 3 4
. . . E           2 3 4 5   <- E에 도달한 거리 = 5

큐(Queue)의 변화 — 거리가 같은 칸들이 '층'을 이루며 퍼진다:
step0  push S(0,0)d=0            queue=[ (0,0)d0 ]
step1  pop (0,0) -> push 우/하   queue=[ (0,1)d1, (1,0)d1 ]
step2  pop (0,1) -> push (0,2)   queue=[ (1,0)d1, (0,2)d2 ]
step3  pop (1,0) -> push (2,0)   queue=[ (0,2)d2, (2,0)d2 ]
...    (거리 0칸 -> 1칸 -> 2칸 순서로 절대 뒤섞이지 않음)
```
핵심: 먼저 큐에 들어간 칸은 더 가깝다. 그래서 처음 도착했을 때의 거리가 곧 최단 거리다.

### (2) 다익스트라 — 가중치 그래프 (양수)
정점 1에서 출발. `dist[]`는 현재까지 알아낸 최단 거리(처음엔 ∞). 우선순위 큐에서 **가장 가까운 정점을 꺼내 확정**하고, 거기서 갈 수 있는 이웃을 "완화(relax)"한다.

```
그래프 (가중치):           1
                      (1)---->(2)
                       | \      |
                     (4)  (2)   (5)
                       v    v    v
                      (3)<-(4)<-(2)   ... 1->2:1, 1->3:4, 1->4:2, 2->4:5, 4->3:1

dist 배열 변화 (∞ = 아직 모름, [확정]):
초기:     dist = [ 1:0   2:INF 3:INF 4:INF ]   PQ={(0,1)}
1 확정:   1에서 2(1),3(4),4(2) 완화
          dist = [ 1:0*  2:1   3:4   4:2   ]   PQ={(1,2),(2,4),(4,3)}
2 확정:   PQ 최솟값=2(거리1). 2->4 완화시 1+5=6 > 2 라 갱신 안 함
          dist = [ 1:0*  2:1*  3:4   4:2   ]   PQ={(2,4),(4,3)}
4 확정:   PQ 최솟값=4(거리2). 4->3 완화: 2+1=3 < 4  => 갱신!
          dist = [ 1:0*  2:1*  3:3   4:2*  ]   PQ={(3,3),(4,3 낡음)}
3 확정:   PQ 최솟값=3(거리3) -> 확정
          dist = [ 1:0*  2:1*  3:3*  4:2*  ]   완료

* = 최단 거리 확정됨. 한 번 확정되면 다시 줄어들지 않는다(양수 간선 보장).
```
주목: 3번 정점은 처음에 4로 들어왔지만, 4번을 거치는 경로(2+1=3)가 더 짧아서 **갱신(완화)**되었다.

### (3) 플로이드-워셜 — 경유지 k를 늘려가며 전체 쌍 갱신
`dist[i][j]`를 "정점 1..k만 경유해도 될 때의 i->j 최단 거리"로 정의하고 k를 1씩 늘린다.

```
점화식:  dist[i][j] = min( dist[i][j], dist[i][k] + dist[k][j] )
         "i->j 직접 가는 것"  vs  "i->k 갔다가 k->j 가는 것" 중 짧은 쪽

초기 (간선만):        k=1 경유 후:         k=2 경유 후:
    1  2  3              1  2  3              1  2  3
1 [ 0  3  ∞ ]        1 [ 0  3  ∞ ]        1 [ 0  3  4 ]  <- 1->2->3 = 3+1=4 발견!
2 [ ∞  0  1 ]        2 [ ∞  0  1 ]        2 [ ∞  0  1 ]
3 [ ∞  ∞  0 ]        3 [ ∞  ∞  0 ]        3 [ ∞  ∞  0 ]
```

## C++ 예제 코드

### 다익스트라 (가장 자주 출제, 우선순위 큐 버전)
```cpp
#include <bits/stdc++.h>
using namespace std;

const long long INF = 1e18;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    int V, E, start;       // 정점 수, 간선 수, 시작 정점
    cin >> V >> E >> start;

    // 인접 리스트: graph[u] = { {목적지 v, 가중치 w}, ... }
    vector<vector<pair<int,int>>> graph(V + 1);
    for (int i = 0; i < E; i++) {
        int u, v, w;
        cin >> u >> v >> w;
        graph[u].push_back({v, w});   // 방향 그래프. 무방향이면 반대도 추가
    }

    vector<long long> dist(V + 1, INF);

    // 최소 힙: (거리, 정점). 거리가 작은 것이 먼저 나오도록 greater 사용
    priority_queue<pair<long long,int>,
                   vector<pair<long long,int>>,
                   greater<>> pq;

    dist[start] = 0;
    pq.push({0, start});

    while (!pq.empty()) {
        auto [d, u] = pq.top();
        pq.pop();

        // 핵심 1: 이미 더 짧은 거리로 확정된 정점이면 건너뛴다(낡은 항목 제거)
        if (d > dist[u]) continue;

        // u에서 갈 수 있는 모든 이웃을 완화(relax)
        for (auto [v, w] : graph[u]) {
            if (dist[u] + w < dist[v]) {   // 더 짧은 경로를 발견하면
                dist[v] = dist[u] + w;     // 거리 갱신
                pq.push({dist[v], v});     // 큐에 다시 넣음
            }
        }
    }

    // 결과 출력: 도달 불가면 INF
    for (int i = 1; i <= V; i++) {
        if (dist[i] == INF) cout << "INF\n";
        else                cout << dist[i] << "\n";
    }
    return 0;
}
```

### BFS 최단 거리 (격자 미로, 모든 이동 비용 1)
```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int n, m;                 // 격자 크기 n x m
    cin >> n >> m;
    vector<string> grid(n);
    for (auto& row : grid) cin >> row;   // 예: "0"=벽, "1"=길

    vector<vector<int>> dist(n, vector<int>(m, -1));  // -1 = 미방문
    queue<pair<int,int>> q;

    dist[0][0] = 1;           // 출발점(문제에 따라 0 또는 1로 시작)
    q.push({0, 0});

    int dx[] = {0, 0, 1, -1}; // 우, 좌, 하, 상
    int dy[] = {1, -1, 0, 0};

    while (!q.empty()) {
        auto [x, y] = q.front();
        q.pop();
        for (int dir = 0; dir < 4; dir++) {
            int nx = x + dx[dir], ny = y + dy[dir];
            // 범위 안 && 길('1') && 아직 방문 안 함
            if (nx < 0 || ny < 0 || nx >= n || ny >= m) continue;
            if (grid[nx][ny] != '1' || dist[nx][ny] != -1) continue;
            dist[nx][ny] = dist[x][y] + 1;   // 한 칸 = +1
            q.push({nx, ny});
        }
    }
    cout << dist[n-1][m-1] << "\n";   // 도착점까지 최단 거리(-1이면 도달 불가)
    return 0;
}
```

### 플로이드-워셜 (전체 쌍 최단 거리, V가 작을 때)
```cpp
#include <bits/stdc++.h>
using namespace std;
const long long INF = 1e18;

int main() {
    int V, E;
    cin >> V >> E;
    vector<vector<long long>> dist(V + 1, vector<long long>(V + 1, INF));
    for (int i = 1; i <= V; i++) dist[i][i] = 0;   // 자기 자신은 0

    for (int i = 0; i < E; i++) {
        int u, v, w; cin >> u >> v >> w;
        dist[u][v] = min(dist[u][v], (long long)w); // 중복 간선은 최솟값
    }

    // 핵심: 경유지 k가 가장 바깥 루프여야 한다!
    for (int k = 1; k <= V; k++)
        for (int i = 1; i <= V; i++)
            for (int j = 1; j <= V; j++)
                if (dist[i][k] != INF && dist[k][j] != INF)   // 오버플로 방지
                    dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j]);

    for (int i = 1; i <= V; i++) {
        for (int j = 1; j <= V; j++)
            cout << (dist[i][j] == INF ? -1 : dist[i][j]) << " ";
        cout << "\n";
    }
    return 0;
}
```

## 시간복잡도
- **BFS**: O(V + E). 모든 정점과 간선을 한 번씩만 본다. 격자에서는 O(행 × 열).
- **다익스트라 (우선순위 큐)**: O(E log V). 각 간선마다 큐 삽입(log V)이 일어난다. 인접 행렬 + 배열 버전은 O(V²)로, 조밀 그래프에선 오히려 이 쪽이 빠를 수 있다.
- **벨만-포드**: O(V · E). 간선 전체를 V-1번 완화. V번째에도 갱신되면 음수 사이클 존재.
- **플로이드-워셜**: O(V³). 3중 for문. V ≤ 400~500 정도까지 현실적. 메모리는 O(V²).

## 흔한 실수 / 팁
- **다익스트라에 음수 간선을 넣지 말 것.** 양수만 보장될 때만 정답. 음수가 있으면 벨만-포드, 음수 사이클까지 탐지하려면 V번째 완화 검사.
- **`dist`를 갱신할 때만 큐에 push** 하고, 꺼낼 때 `if (d > dist[u]) continue;`로 낡은 항목을 버려야 한다. 이걸 빼면 시간초과가 난다(같은 정점이 여러 번 처리됨).
- **INF 오버플로 주의.** `dist[u] + w`가 INF + 양수로 넘칠 수 있다. INF를 `1e18`처럼 적당히 크되 덧셈해도 오버플로 안 나는 값으로, 또는 갱신 전에 `dist[u] != INF` 검사를 넣는다. 플로이드-워셜에선 `int` 합산 오버플로가 특히 잦으니 `long long`을 쓰자.
- **자료형은 `long long`** 을 기본으로. 간선 가중치와 정점 수가 크면 거리 합이 int 범위(약 21억)를 쉽게 넘는다.
- **BFS는 가중치가 전부 같을 때만** 최단 경로를 보장한다. 가중치가 0과 1만 섞여 있다면 deque를 쓰는 **0-1 BFS**(가중치 0이면 앞에, 1이면 뒤에 push)로 O(V+E)에 풀 수 있다.
- **플로이드-워셜은 k(경유지) 루프가 반드시 가장 바깥.** i, j를 바깥에 두면 틀린 답이 나온다. 흔한 실수 1순위.
- **무방향 그래프**는 간선 입력 시 양방향 모두 추가해야 한다(`u->v`와 `v->u`).
- **경로 자체를 복원**해야 하면 `parent[v] = u`를 완화할 때 함께 기록하고, 도착점에서 출발점까지 거꾸로 따라가 뒤집는다.
- **방문 처리 시점:** BFS는 큐에 **넣을 때** 방문 표시해야 한다(꺼낼 때 표시하면 같은 칸이 중복으로 큐에 들어가 비효율/오류 발생).
