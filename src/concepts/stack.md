# 스택 (Stack)

## 한 줄 요약
**나중에 넣은 것이 먼저 나오는(LIFO, Last-In-First-Out)** 자료구조. 한쪽 끝에서만 넣고(push) 뺀다(pop).

## 언제 쓰나
- 괄호 짝 맞추기, 수식 계산
- "직전 것"을 되돌아봐야 할 때 (실행 취소, 뒤로 가기)
- DFS(깊이 우선 탐색)의 호출 스택, 재귀의 비재귀 변환
- 모노토닉 스택 (다음 큰 수 찾기 등)

## 왜 이거 + 대안 + 차이
- 큐(Queue)는 **FIFO** (먼저 넣은 것이 먼저 나옴). "순서대로 처리"는 큐, "가장 최근 것부터"는 스택.
- 배열로도 흉내낼 수 있지만, 스택은 의도("끝에서만 접근")를 코드로 드러내 실수를 줄인다.
- C++에서는 `std::stack<T>` (내부적으로 deque). push/pop/top/empty/size 모두 O(1).

## 그림으로 보기

```
push(A) push(B) push(C)        pop() → C 반환
  ┌───┐   ┌───┐   ┌───┐          ┌───┐
  │   │   │   │   │ C │ ← top    │   │
  │   │   │ B │   │ B │          │ B │ ← top
  │ A │   │ A │   │ A │          │ A │
  └───┘   └───┘   └───┘          └───┘
```

## 짧은 예제

```cpp
#include <stack>
using namespace std;

stack<int> st;
st.push(1);
st.push(2);
int x = st.top();   // 2 (맨 위 확인)
st.pop();           // 2 제거 (pop 은 값을 반환하지 않음에 주의!)
bool e = st.empty(); // false
```

## 괄호 검사에 쓰는 이유
여는 괄호를 만나면 쌓아두고, 닫는 괄호를 만나면 가장 최근에 쌓은 여는 괄호와 짝짓는다.
"가장 최근 것" 을 꺼내야 하므로 LIFO인 스택이 딱 맞는다.
(괄호 종류가 한 가지뿐이면 카운터 정수 하나로도 대체 가능 — 스택의 특수한 경우다.)

## 흔한 실수
- `pop()` 은 값을 돌려주지 않는다. 값이 필요하면 `top()` 으로 먼저 읽고 `pop()`.
- 비어 있는 스택에 `top()`/`pop()` 호출 → 정의되지 않은 동작. 항상 `empty()` 확인.
