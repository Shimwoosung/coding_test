# 스택 (Stack)

## 한 줄 요약
스택은 "나중에 넣은 것이 가장 먼저 나오는(LIFO, Last In First Out)" 구조로, 한쪽 끝(top)에서만 넣고(push) 빼는(pop) 자료구조다. 마치 접시를 쌓아 올렸다가 위에서부터 꺼내는 것과 같다.

---

## 언제 쓰나
실제 코딩테스트에서 스택이 자연스럽게 떠올라야 하는 문제 유형들이다.

- **괄호 짝 맞추기 / 괄호 유효성 검사** — `(()[])` 같은 문자열이 올바른지 검사 (백준 9012 등)
- **수식 변환·계산** — 중위 표기식 → 후위 표기식(postfix) 변환, 후위 표기식 계산기
- **되돌리기(Undo) / 뒤로 가기** — 브라우저 방문 기록, 에디터의 Ctrl+Z
- **DFS(깊이 우선 탐색)** — 재귀 대신 명시적 스택으로 구현
- **모노토닉 스택(Monotonic Stack)** — "오큰수(다음 큰 수) 찾기", "히스토그램에서 가장 큰 직사각형", "주식 가격" 등 "각 원소의 오른쪽/왼쪽에서 처음 만나는 더 큰/작은 값" 류 문제
- **함수 호출 관리** — 컴퓨터가 함수를 호출할 때 쓰는 콜 스택(call stack) 자체가 스택이다
- **문자열 압축/제거** — 연속 중복 문자 제거(LeetCode "Remove Adjacent Duplicates")

> 핵심 신호: "가장 최근 것부터 처리해야 한다", "짝을 맞춰야 한다", "직전 상태로 되돌아간다" → 스택을 의심하라.

---

## 왜 이걸 쓰나 + 대안과 차이

스택의 본질은 **"순서를 뒤집어서 가장 최근 것에 즉시 접근"**하는 것이다. 다른 자료구조와 비교하면 왜 스택을 골라야 하는지가 분명해진다.

| 자료구조 | 꺼내는 순서 | 어디서 넣고 빼나 | 대표 용도 |
|---|---|---|---|
| **스택 (Stack)** | LIFO (마지막 것 먼저) | 한쪽 끝(top)에서만 | 괄호, DFS, Undo |
| **큐 (Queue)** | FIFO (먼저 넣은 것 먼저) | 한쪽 넣고 반대쪽 뺌 | BFS, 작업 대기열 |
| **덱 (Deque)** | 양쪽 자유롭게 | 앞·뒤 모두 | 슬라이딩 윈도우 |
| **배열 (Array)** | 임의 인덱스 접근 | 어디든 (단, 중간 삽입 O(N)) | 랜덤 접근이 필요할 때 |

- **vs 큐**: 둘 다 "끝에서 넣고 끝에서 빼는" 선형 구조지만 방향이 정반대다. BFS는 큐, DFS는 스택을 쓴다. "가장 먼저 온 것"을 처리하면 큐, "가장 늦게 온 것"을 처리하면 스택.
- **vs 재귀**: DFS를 재귀로 짜면 코드가 짧지만 깊이가 깊을 때 콜 스택이 터진다(스택 오버플로). 명시적 스택으로 바꾸면 깊이 제한에서 자유롭다. 사실 재귀 = 시스템이 대신 관리해 주는 스택이다.
- **vs 그냥 반복문 탐색**: "각 원소마다 오른쪽에서 자기보다 큰 첫 값"을 이중 반복문으로 찾으면 O(N²)이다. 모노토닉 스택을 쓰면 각 원소가 스택에 한 번 들어가고 한 번 나오므로 **O(N)**으로 줄어든다. 이게 스택의 가장 강력한 응용이다.

## 동작 과정

### 1) 기본 push / pop (LIFO 시각화)

`1, 2, 3`을 차례로 push한 뒤 pop을 두 번 하는 과정. 스택은 위(top)로 자란다.

```
초기 상태        push(1)        push(2)        push(3)
                                  
 (비어있음)       +---+          +---+          +---+ <- top
                  | 1 | <-top    | 2 | <-top    | 3 |
                  +---+          +---+          +---+
                                 | 1 |          | 2 |
                                 +---+          +---+
                                                | 1 |
                                                +---+
                  top=1          top=2          top=3 (size 3)


pop() -> 3 반환                 pop() -> 2 반환
+---+                           +---+ <- top
| 2 | <- top                    | 1 |
+---+                           +---+
| 1 |
+---+
size 2                          size 1

핵심: 가장 마지막에 넣은 3이 가장 먼저 나온다 (LIFO).
```

### 2) 괄호 검사 동작 추적

문자열 `( [ ] )` 를 검사한다. 여는 괄호는 push, 닫는 괄호는 top과 짝이 맞는지 확인하며 pop.

```
입력:  (  [  ]  )
        ^                 스택: [ (              ( push
           ^              스택: [ ( , [          [ push
              ^           스택: [ (              ] 만남 -> top '[' 와 짝 OK -> pop
                 ^        스택: [ ] (비어있음)   ) 만남 -> top '(' 와 짝 OK -> pop

끝까지 처리 후 스택이 비어있음 => "올바른 괄호열" (YES)
```

잘못된 예 `( ]`:

```
입력:  (  ]
        ^                 스택: [ (              ( push
           ^              스택: [ (              ] 만남 -> top '(' 와 짝 안 맞음! => "NO"
```

### 3) 모노토닉 스택 — "오큰수(다음에 나오는 더 큰 수)" 찾기

배열 `[3, 5, 2, 7]` 에서 각 원소의 오른쪽에 처음 나오는 더 큰 수를 구한다. 스택에는 "아직 답을 못 찾은 원소의 인덱스"를 담는다.

```
배열:   index  0  1  2  3
        value  3  5  2  7
정답:          ?  ?  ?  ?

i=0 (3): 스택 비어있음 -> 인덱스 0 push.        스택(값): [3]
i=1 (5): top=3 < 5  -> 0의 오큰수는 5! pop.      ans[0]=5
         스택 비어있음 -> 1 push.                 스택(값): [5]
i=2 (2): top=5 > 2  -> 그대로 둠. 2 push.        스택(값): [5, 2]
i=3 (7): top=2 < 7  -> 2의 오큰수는 7! pop.      ans[2]=7
         top=5 < 7  -> 5의 오큰수는 7! pop.      ans[1]=7
         스택 비어있음 -> 3 push.                 스택(값): [7]
끝. 스택에 남은 3(값 7)은 오큰수 없음 -> ans[3]=-1

최종 정답: [5, 7, 7, -1]
```

각 원소는 스택에 정확히 한 번 들어갔다 한 번 나오므로 전체 O(N)이다.

## C++ 예제 코드

### 예제 A. STL `std::stack` 기본 사용

```cpp
#include <iostream>
#include <stack>
using namespace std;

int main() {
    stack<int> st;

    st.push(1);          // 1 넣기
    st.push(2);          // 2 넣기
    st.push(3);          // 3 넣기  -> top은 3

    cout << st.top() << "\n";   // 3 (맨 위 원소 확인, 제거 X)
    cout << st.size() << "\n";  // 3

    st.pop();            // top(3) 제거
    cout << st.top() << "\n";   // 2

    // 스택을 끝까지 비우며 출력 (LIFO 순서로 나옴)
    while (!st.empty()) {       // 비었는지 항상 확인!
        cout << st.top() << " ";// 현재 top
        st.pop();               // 제거
    }
    cout << "\n";        // 출력: 2 1
    return 0;
}
```

> 주의: `std::stack`의 `pop()`은 **값을 반환하지 않는다**. 값이 필요하면 `top()`으로 먼저 읽고 `pop()`으로 지운다.

### 예제 B. 괄호 유효성 검사 (실전 빈출)

```cpp
#include <iostream>
#include <stack>
#include <string>
using namespace std;

bool isValid(const string& s) {
    stack<char> st;
    for (char c : s) {
        if (c == '(' || c == '[' || c == '{') {
            st.push(c);                 // 여는 괄호는 그냥 쌓는다
        } else {
            // 닫는 괄호인데 스택이 비었으면 짝이 없는 것 -> 실패
            if (st.empty()) return false;
            char top = st.top();
            // 짝이 맞는지 검사
            if ((c == ')' && top != '(') ||
                (c == ']' && top != '[') ||
                (c == '}' && top != '{')) {
                return false;           // 짝 불일치
            }
            st.pop();                   // 짝이 맞으면 여는 괄호 제거
        }
    }
    return st.empty();   // 다 처리하고 남은 게 없어야 올바른 괄호열
}

int main() {
    cout << (isValid("([])") ? "YES" : "NO") << "\n"; // YES
    cout << (isValid("(]")   ? "YES" : "NO") << "\n"; // NO
    cout << (isValid("(((")  ? "YES" : "NO") << "\n"; // NO (안 닫힘)
    return 0;
}
```

### 예제 C. 모노토닉 스택 — 오큰수 구하기 (O(N))

```cpp
#include <iostream>
#include <vector>
#include <stack>
using namespace std;

int main() {
    vector<int> a = {3, 5, 2, 7};
    int n = a.size();
    vector<int> ans(n, -1);   // 기본값 -1 (오큰수 없음)
    stack<int> st;            // "아직 답을 못 찾은 원소의 인덱스" 저장

    for (int i = 0; i < n; i++) {
        // 현재 값 a[i]가 스택 top이 가리키는 값보다 크면,
        // 그 원소들의 오큰수가 바로 a[i]다.
        while (!st.empty() && a[st.top()] < a[i]) {
            ans[st.top()] = a[i];  // 답 확정
            st.pop();              // 처리 끝난 인덱스 제거
        }
        st.push(i);                // 현재 인덱스를 후보로 넣기
    }
    // 스택에 남은 인덱스들은 오큰수가 없으므로 -1 그대로 둔다.

    for (int x : ans) cout << x << " ";  // 5 7 7 -1
    cout << "\n";
    return 0;
}
```

### 예제 D. 배열로 직접 스택 구현 (내부 원리 이해용)

```cpp
#include <iostream>
using namespace std;

const int MAX = 100000;
int data[MAX];
int topIdx = -1;            // 비어있을 때 -1, 가장 위 원소의 인덱스

void push(int x) { data[++topIdx] = x; }   // 인덱스 늘리고 저장
int  top()       { return data[topIdx]; }  // 현재 맨 위
void pop()       { --topIdx; }             // 인덱스만 줄이면 끝
bool empty()     { return topIdx == -1; }

int main() {
    push(10); push(20); push(30);
    cout << top() << "\n";   // 30
    pop();
    cout << top() << "\n";   // 20
    return 0;
}
```

> 이렇게 `top` 인덱스 하나만 움직이면 push/pop이 모두 O(1)임을 알 수 있다.

## 시간복잡도

| 연산 | 시간복잡도 | 설명 |
|---|---|---|
| `push` (넣기) | **O(1)** | top에 추가만 하면 됨 |
| `pop` (빼기) | **O(1)** | top만 제거 |
| `top` / `peek` (확인) | **O(1)** | 인덱스 한 번 접근 |
| `empty` / `size` | **O(1)** | 상태 변수 확인 |

- **공간복잡도**: 저장한 원소 수에 비례하여 O(N).
- **모노토닉 스택 전체**: 원소마다 push 1회 + pop 최대 1회 → 전체 **O(N)** (각 연산이 O(1)이고 총 연산 횟수가 2N 이하라는 점이 핵심, "분할 상환(amortized)" 분석).

## 흔한 실수 / 팁

1. **비어있는 스택에서 top/pop 호출** — 가장 흔한 런타임 에러(혹은 쓰레기값). `top()`이나 `pop()` 전에는 **반드시 `!st.empty()`를 먼저 확인**하라. 특히 괄호 검사·모노토닉 스택의 `while` 조건에서 `!st.empty()`를 앞에 두는 것이 중요하다(단락 평가 활용).

   ```cpp
   while (!st.empty() && a[st.top()] < a[i]) // OK: empty 검사가 먼저
   while (a[st.top()] < a[i] && !st.empty()) // 위험: 빈 스택에서 st.top() 접근!
   ```

2. **`std::stack::pop()`은 값을 반환하지 않는다** — `int x = st.pop();` 은 컴파일 에러다. `int x = st.top(); st.pop();` 으로 분리해야 한다.

3. **DFS 깊이가 깊을 때 재귀는 스택 오버플로** — 정점이 수십만 개로 길게 이어지는 그래프에서 재귀 DFS는 콜 스택이 터질 수 있다. 명시적 `std::stack`으로 바꾸거나, 환경에 따라 스택 크기를 늘려라.

4. **모노토닉 스택에는 값이 아니라 인덱스를 저장하라** — 답을 채울 위치(`ans[그 인덱스]`)를 알아야 하므로 값만 넣으면 어디에 답을 쓸지 잃어버린다.

5. **부등호 방향과 등호 처리** — "다음 큰 수"인지 "다음 크거나 같은 수"인지에 따라 `<` 와 `<=` 가 달라진다. 문제 조건을 꼼꼼히 읽고 같은 값일 때 pop할지 말지를 결정하라.

6. **스택과 큐를 헷갈리지 말 것** — "먼저 온 사람부터"면 큐(BFS), "마지막에 온 것부터/짝 맞추기/되돌리기"면 스택(DFS). 방향만 다르다는 점을 항상 떠올려라.

7. **벡터로 스택을 흉내 낼 때** — `vector<int> st; st.push_back(x); st.back(); st.pop_back();` 패턴도 자주 쓴다. 인덱스 기반 접근이 필요하면 오히려 벡터가 편할 때도 있다.
