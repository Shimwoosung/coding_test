# 구현 / 시뮬레이션 (Implementation)

## 한 줄 요약
특별한 알고리즘보다 "문제에 적힌 규칙을 정확히 코드로 옮기는" 능력. 코테에서 가장 흔하다.

## 언제 쓰나
- 격자 채우기/회전 (달팽이, 90도 회전)
- 규칙대로 진행 (더하기 사이클, 지뢰찾기 숫자)
- 작은 범위 완전탐색 (체스판 다시 칠하기)

## 왜 이거 + 대안 + 차이
- 알고리즘적 트릭보다 방향 배열(dx/dy), 범위 체크, 인덱스 관리가 관건.
- 범위가 작으면(8x8 등) 굳이 영리하게 안 하고 전부 시도(브루트포스)가 정답.

## 핵심 도구
- 방향 배열: `int dx[]={0,0,1,-1}, dy[]={1,-1,0,0};`
- 경계 체크 헬퍼, 2차원 인덱스 회전 공식.

## 흔한 실수
- 범위(out of bounds) 체크 누락 → 런타임 에러
- 행/열, x/y 헷갈림. 회전 후 크기(N×M → M×N) 바뀜 주의
