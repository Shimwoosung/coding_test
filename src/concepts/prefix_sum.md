# 누적 합 (Prefix Sum)

## 한 줄 요약
배열의 "앞에서부터 더한 합"을 미리 계산해 두면, 어떤 구간의 합이든 **뺄셈 한 번(O(1))** 으로 즉시 구할 수 있는 기법이다.

---

## 언제 쓰나
- **구간 합을 여러 번 물어보는 문제**: "i번째부터 j번째까지의 합을 구하라"는 질의가 수백~수십만 번 들어올 때.
- **부분 배열의 합이 K인 경우의 수 세기** (해시맵 + 누적 합 조합).
- **2차원 격자에서 직사각형 영역의 합** 구하기 (2D 누적 합).
- **차이 배열(Difference Array)** 과 짝을 이뤄 "구간에 일괄로 값 더하기"를 빠르게 처리할 때.
- **평균/합 기반 조건 판정**: 어떤 길이의 연속 구간이 조건을 만족하는지 빠르게 검사할 때.
- 예시 백준/프로그래머스 유형: 구간 합 구하기, 수들의 합, 부분합, 직사각형 합, 연속 부분 수열의 합 등.

---

## 왜 이걸 쓰나 + 대안과 차이

핵심 문제 상황: 배열에서 구간 `[l, r]`의 합을 구하고 싶다.

| 방법 | 사전 계산 | 질의 1회 비용 | 질의 Q번 총비용 | 값이 자주 바뀌면 |
|------|-----------|---------------|------------------|------------------|
| 매번 직접 더하기 | 없음 | O(N) | O(N·Q) | 문제 없음 |
| **누적 합** | O(N) | **O(1)** | **O(N + Q)** | 매번 재계산 → 비효율 |
| 펜윅 트리(BIT) | O(N) | O(log N) | O(Q log N) | **갱신도 O(log N)** |
| 세그먼트 트리 | O(N) | O(log N) | O(Q log N) | 갱신 O(log N), 범위 넓음 |

- **직접 더하기 대비**: 질의가 많으면 압도적으로 빠르다. N=10만, Q=10만이면 직접 더하기는 100억 연산 → 시간 초과. 누적 합은 약 20만 연산.
- **펜윅/세그먼트 트리 대비**: 누적 합은 **배열이 고정(값 변경 없음)** 일 때 최강이다. 구현이 가장 간단하고 상수도 작다. 하지만 중간에 원소 값이 자주 바뀌면, 바뀔 때마다 누적 합 전체를 다시 만들어야 하므로(O(N)) 손해다. 이때는 펜윅 트리를 쓴다.

요약: **"값은 거의 안 바뀌는데, 구간 합 질문이 엄청 많다"** → 누적 합이 정답.

---

## 동작 과정

원본 배열 `A`가 있을 때, 누적 합 배열 `P`를 다음과 같이 정의한다 (1-indexed 권장).

```
P[0] = 0
P[i] = P[i-1] + A[i]      (i = 1, 2, ..., N)
```

즉 `P[i]`는 "A의 1번째부터 i번째까지의 합"이다.

### 1) 누적 합 배열 만들기

```
원본 A (1-indexed):
 index :   1    2    3    4    5
       +----+----+----+----+----+
   A   |  3 |  1 |  4 |  1 |  5 |
       +----+----+----+----+----+

P[0] = 0
P[1] = P[0] + A[1] = 0 + 3 =  3
P[2] = P[1] + A[2] = 3 + 1 =  4
P[3] = P[2] + A[3] = 4 + 4 =  8
P[4] = P[3] + A[4] = 8 + 1 =  9
P[5] = P[4] + A[5] = 9 + 5 = 14

누적 합 P:
 index :  0    1    2    3    4    5
       +----+----+----+----+----+----+
   P   |  0 |  3 |  4 |  8 |  9 | 14 |
       +----+----+----+----+----+----+
         ^
         경계용 0 (왼쪽 끝을 깔끔히 처리하기 위함)
```

### 2) 구간 합을 뺄셈으로 구하기

구간 `[l, r]`의 합 = `P[r] - P[l-1]`

예: 2번째 ~ 4번째 구간(A[2]+A[3]+A[4] = 1+4+1 = 6)을 구해보자.

```
구하고 싶은 영역:  [l=2, r=4]
       +----+====+====+====+----+
   A   |  3 | 1  | 4  | 1  |  5 |
       +----+====+====+====+----+

P[4] = 9   ← 1번째부터 4번째까지 합 (3+1+4+1)
P[1] = 3   ← 1번째부터 1번째까지 합 (3)      ← l-1 = 1

       P[4]  = [ A1 A2 A3 A4 ]          = 9
     - P[1]  = [ A1          ]          = 3
     ------------------------------------------
       결과  =    [ A2 A3 A4 ]          = 6   ✅

겹치는 앞부분(A1)이 빼지면서 원하는 구간만 정확히 남는다.
```

핵심 직관: `P[r]`은 앞에서 r까지 다 더한 값, `P[l-1]`은 l 직전까지 더한 값. 둘을 빼면 **l부터 r까지만** 남는다. 마치 "큰 막대에서 앞쪽 막대를 잘라내는" 느낌이다.

### 3) 응용 — 부분 배열의 합이 K인 경우의 수 (해시맵)

"연속 구간의 합 = K"는 `P[r] - P[l-1] = K`, 즉 `P[l-1] = P[r] - K`로 바꿀 수 있다. 왼쪽에서 오른쪽으로 가며 지금까지 등장한 누적 합 값들을 해시맵에 세어두면, 각 위치마다 `P[r]-K`가 몇 번 나왔는지로 답을 누적한다.

```
A = [1, 2, 3], K = 3 인 경우

진행하며 P 값과 등장 횟수(map)를 관리:
초기: map = { 0:1 }   (빈 구간의 누적합 0을 미리 1개 넣어둠)

P=1 일 때: 찾는 값 = 1-3 = -2 → map에 없음(0개)   map={0:1, 1:1}
P=3 일 때: 찾는 값 = 3-3 =  0 → map에 1개! 정답 +1   map={0:1,1:1,3:1}
P=6 일 때: 찾는 값 = 6-3 =  3 → map에 1개! 정답 +1   map={...,6:1}

총 2개  → 구간 [1..2]=(1+2)=3,  구간 [3..3]=3  ✅
```

## C++ 예제 코드

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    int N, Q;
    cin >> N >> Q;                 // 원소 개수 N, 질의 개수 Q

    // 1-indexed로 입력받는다. A[0]은 사용하지 않음.
    vector<long long> A(N + 1);
    for (int i = 1; i <= N; i++) cin >> A[i];

    // 누적 합 배열. 크기 N+1, P[0] = 0 으로 경계 처리.
    // long long을 쓰는 이유: 합이 int 범위(약 21억)를 넘기 쉽기 때문.
    vector<long long> P(N + 1, 0);
    for (int i = 1; i <= N; i++) {
        P[i] = P[i - 1] + A[i];   // 점화식: 직전 누적합 + 현재 원소
    }

    // 각 질의 [l, r]의 구간 합을 O(1)로 응답
    while (Q--) {
        int l, r;
        cin >> l >> r;
        long long sum = P[r] - P[l - 1];   // 핵심 한 줄!
        cout << sum << '\n';
    }
    return 0;
}
```

### (보너스) 2차원 누적 합 — 직사각형 영역의 합

```cpp
// S[i][j] = (1,1)부터 (i,j)까지의 사각형 합
// 점화식(포함-배제 원리):
//   S[i][j] = A[i][j] + S[i-1][j] + S[i][j-1] - S[i-1][j-1];
//
// (r1,c1) ~ (r2,c2) 직사각형 합:
//   S[r2][c2] - S[r1-1][c2] - S[r2][c1-1] + S[r1-1][c1-1];
//
// 그림(겹치는 부분을 두 번 빼므로 한 번 더해줌):
//   ┌───────────┐
//   │  A  │  B   │   원하는 영역 = 전체 - 위(B+A부분) - 왼(C+A부분) + A(두번빠진것)
//   ├─────┼──────┤
//   │  C  │ 영역 │
//   └─────┴──────┘
```

## 시간복잡도

| 단계 | 복잡도 |
|------|--------|
| 누적 합 배열 만들기 (전처리) | O(N) |
| 구간 합 질의 1회 | **O(1)** |
| 질의 Q회 전체 | O(N + Q) |
| 2차원 누적 합 전처리 | O(N·M) |
| 2차원 구간 합 질의 1회 | O(1) |
| 공간복잡도 | O(N) (또는 2D는 O(N·M)) |

---

## 흔한 실수 / 팁

1. **인덱스 경계 (l-1) 실수**: `P[r] - P[l-1]`에서 `l`이 1일 때 `P[0]`을 참조한다. 그래서 **1-indexed + P[0]=0** 조합을 강력 추천. 0-indexed로 하면 `l==0`일 때 `P[-1]` 접근으로 터지기 쉽다.
2. **오버플로우**: 합이 커지면 `int`로는 부족하다. 누적 합은 거의 항상 **`long long`**으로 잡아라. (예: 원소 10만 개 × 각 10억 = 10^14, int 불가)
3. **0-indexed vs 1-indexed 혼동**: 한 코드 안에서 섞으면 off-by-one 버그의 온상이다. 하나로 통일하라. 1-indexed가 누적 합과 가장 궁합이 좋다.
4. **값이 바뀌는 문제에 무리하게 쓰기**: 원소가 중간에 갱신되는 문제라면 누적 합을 매번 새로 만드는 순간 O(N)이 되어 의미가 없다 → **펜윅 트리(BIT)** 로 전환하라.
5. **차이 배열과의 콤보 기억하기**: "구간 [l,r]에 +x를 여러 번"은 차이 배열 `D[l]+=x, D[r+1]-=x` 후 마지막에 누적 합 한 번으로 복원하면 O(N+Q). 누적 합의 "역방향" 친구로 같이 외워두면 좋다.
6. **빠른 입출력**: 질의가 많은 문제이므로 C++에선 `sync_with_stdio(false); cin.tie(NULL);`을 꼭 넣어 시간 초과를 피하자.
7. **2D는 포함-배제 부호 주의**: `- 위 - 왼 + 대각선`. 부호 하나만 틀려도 답이 어긋난다. 작은 예제로 손으로 검산하는 습관을 들이자.
