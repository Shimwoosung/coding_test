# 큐 (Queue) & BFS

## 한 줄 요약
큐는 **먼저 들어온 데이터가 먼저 나가는(FIFO, First In First Out)** 선형 자료구조이며, 그래프/격자 탐색의 핵심인 **BFS(너비 우선 탐색)** 의 엔진이다.

## 언제 쓰나
- **최단 거리/최소 횟수 문제**: 가중치가 없는(=한 칸 이동 비용이 모두 1인) 그래프나 격자에서 "출발점 → 도착점 최소 이동 칸 수", "최소 변환 횟수" 등을 구할 때. (예: 미로 탈출, 토마토 익히기, 숨바꼭질, 단지번호붙이기)
- **레벨 단위 처리**: "한 번에 동시에 퍼지는" 시뮬레이션. (예: 전염병 확산, 불 번지기, 물/벽 동시 시뮬레이션)
- **순서 보장이 필요한 작업 처리**: 들어온 순서대로 처리해야 하는 작업 대기열, 프린터 큐, 캐시, 메시지 큐.
- **상태 공간 탐색**: 현재 상태에서 도달 가능한 다음 상태들을 "가까운 것부터" 펼쳐 나가야 할 때. (예: 연산자로 숫자 만들기, 퍼즐 최소 조작)

## 왜 이걸 쓰나 + 대안과 차이
핵심은 **"꺼내는 순서"** 다. 자료구조마다 꺼내는 규칙이 다르고, 그 규칙이 탐색의 성격을 결정한다.

| 자료구조 | 꺼내는 순서 | 대표 알고리즘 | 특징 |
|----------|-------------|----------------|------|
| **큐 (Queue)** | 먼저 들어온 것 (FIFO) | **BFS** | 출발점에서 **가까운 곳부터** 동심원처럼 퍼짐 → 가중치 1 그래프에서 최단거리 보장 |
| **스택 (Stack)** | 나중에 들어온 것 (LIFO) | DFS | 한 방향으로 **끝까지 파고든 뒤** 되돌아옴 → 최단거리 보장 안 됨 |
| **우선순위 큐 (Priority Queue)** | 가장 작은(또는 큰) 값 | 다익스트라 | 간선 **가중치가 다를 때** 최단거리. 큐보다 느림 (log n) |

- **BFS vs DFS**: 둘 다 "모든 정점을 방문"할 수 있지만, **최단 거리**가 필요하면 BFS를 써야 한다. DFS는 처음 도달한 경로가 최단이라는 보장이 없다. 반대로 "경로 자체를 모두 탐색", "백트래킹", "사이클 판정" 등은 DFS가 자연스럽다.
- **BFS vs 다익스트라**: 이동 비용이 전부 같으면(예: 한 칸 = 1) 굳이 우선순위 큐를 쓸 필요 없이 **일반 큐 BFS면 충분하고 더 빠르다**. 비용이 칸마다 다르면 다익스트라(우선순위 큐)가 필요하다.
- **왜 FIFO가 최단거리를 보장하나?** 큐는 "거리 0짜리"를 모두 처리한 뒤 "거리 1짜리"를 처리하고, 그다음 "거리 2짜리"를 처리한다. 즉 **거리가 작은 순서대로** 정점을 확정하므로, 어떤 정점에 처음 도달했을 때의 거리가 곧 최단 거리가 된다.

## 동작 과정

### (1) 큐의 기본 push / pop (FIFO)
오른쪽(rear)으로 넣고(push), 왼쪽(front)에서 뺀다(pop).

```
초기:        front -> [   ] <- rear        (비어 있음)

push(10):    front -> [10] <- rear
push(20):    front -> [10][20] <- rear
push(30):    front -> [10][20][30] <- rear

pop() -> 10  front -> [20][30] <- rear      (가장 먼저 들어온 10이 나감)
pop() -> 20  front -> [30] <- rear
push(40):    front -> [30][40] <- rear
pop() -> 30  front -> [40] <- rear
```
스택이었다면 pop 순서가 30, 20, 10 이었을 것. 큐는 들어온 순서 그대로 10, 20, 30.

### (2) 격자 BFS — 동심원처럼 퍼지는 모습
시작점 S에서 한 칸씩 퍼진다. 숫자는 시작점으로부터의 **거리(=BFS 레벨)** 다.

```
격자 (.=빈칸, #=벽, S=시작)        거리 채워지는 과정 (BFS 레벨)

  S . . .                          0 1 2 3
  . # . .            ===>          1 # 3 4
  . . . #                          2 3 4 #
  # . . .                          # 4 5 6

레벨별 큐 상태 (좌표를 거리 순으로 꺼냄):
  큐: [S(0)]                       <- 거리 0 처리
  큐: [(0,1)=1, (1,0)=1]           <- 거리 1짜리들이 한꺼번에 큐에 들어감
  큐: [(0,2)=2, (2,0)=2]           <- 거리 2
  ...                              가까운 칸부터 차례로 확정 => 최단거리!
```
포인트: **방문 표시(visited)를 큐에 "넣는 순간"** 찍어야 같은 칸이 큐에 중복으로 들어가지 않는다.

### (3) 한 칸 확장 시 큐 변화 추적
```
방향 = 상하좌우 4방향, dist[][]는 거리 배열

현재 front에서 (1,1)을 꺼냈다고 하자. dist[1][1] = 2.
  -> 이웃 (0,1),(2,1),(1,0),(1,2) 검사
  -> 아직 방문 안 했고 벽 아닌 칸만:
       dist[이웃] = dist[1][1] + 1 = 3
       visited[이웃] = true
       큐.push(이웃)

큐:  앞[ ... ][(0,1)][(2,1)][(1,2)]뒤   <- 새 이웃이 뒤(rear)에 붙음
```

## C++ 예제 코드

### 예제 A — `std::queue` 기본 사용법
```cpp
#include <iostream>
#include <queue>
using namespace std;

int main() {
    queue<int> q;

    q.push(10);            // 뒤(rear)로 삽입
    q.push(20);
    q.push(30);

    cout << q.front() << "\n";  // 10  (가장 먼저 들어온 원소)
    cout << q.back()  << "\n";  // 30  (가장 마지막에 들어온 원소)
    cout << q.size()  << "\n";  // 3

    while (!q.empty()) {        // 비어 있는지 항상 검사하고 접근
        cout << q.front() << " ";  // 10 20 30 순서
        q.pop();                   // pop()은 값을 "제거만" 한다(반환 X)
    }
    cout << "\n";
    return 0;
}
```

### 예제 B — 격자 BFS로 최단 거리 구하기 (미로 탈출의 표준 형태)
```cpp
#include <iostream>
#include <queue>
#include <cstring>
using namespace std;

int n, m;                 // 행, 열
int grid[1000][1000];     // 1 = 이동 가능, 0 = 벽
int dist[1000][1000];     // 시작점으로부터의 최단 거리(-1 = 아직 미방문)

// 상, 하, 좌, 우 4방향 이동 벡터
int dx[4] = {-1, 1, 0, 0};
int dy[4] = {0, 0, -1, 1};

int bfs(int sx, int sy) {
    // dist를 -1로 초기화 → -1이면 "아직 방문 안 함" 표시 겸용
    memset(dist, -1, sizeof(dist));

    queue<pair<int,int>> q;   // (행, 열) 좌표를 담는 큐
    q.push({sx, sy});
    dist[sx][sy] = 0;         // 시작점 거리 = 0, 동시에 방문 처리

    while (!q.empty()) {
        auto [x, y] = q.front();  // 가장 먼저 들어온(=가장 가까운) 칸을 꺼냄
        q.pop();

        for (int d = 0; d < 4; d++) {
            int nx = x + dx[d];
            int ny = y + dy[d];

            // 1) 격자 밖이면 무시
            if (nx < 0 || ny < 0 || nx >= n || ny >= m) continue;
            // 2) 벽이거나 이미 방문한 칸이면 무시
            if (grid[nx][ny] == 0 || dist[nx][ny] != -1) continue;

            // 3) 처음 도달 → 거리 = 현재 거리 + 1, 큐에 넣으며 방문 처리
            dist[nx][ny] = dist[x][y] + 1;
            q.push({nx, ny});
        }
    }
    // 예: 우하단 도착점까지 최단 거리 반환 (도달 불가면 -1)
    return dist[n-1][m-1];
}

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    cin >> n >> m;
    for (int i = 0; i < n; i++)
        for (int j = 0; j < m; j++)
            cin >> grid[i][j];

    cout << bfs(0, 0) << "\n";
    return 0;
}
```

### 예제 C — 레벨(단계) 단위로 끊어서 처리하기
"몇 단계 만에 전부 퍼지는가" 같은 문제에서 자주 쓰는 패턴.
```cpp
int levels = 0;
while (!q.empty()) {
    int sz = q.size();          // 현재 레벨에 있는 원소 개수를 "고정"
    for (int i = 0; i < sz; i++) {   // 딱 이번 레벨만큼만 꺼낸다
        auto [x, y] = q.front();
        q.pop();
        // ... 이웃을 큐에 push (다음 레벨로 쌓임)
    }
    levels++;                    // 한 레벨(한 단계) 완료
}
```

## 시간복잡도
- **큐 연산**: `push`, `pop`, `front`, `empty` 모두 **O(1)**.
- **BFS 전체**: 정점 V개, 간선 E개일 때 **O(V + E)**.
  - 각 정점은 큐에 **딱 한 번씩만** 들어가고 나온다(visited 덕분).
  - 각 간선(이웃 검사)은 상수 번 확인된다.
- **격자 BFS**: 칸 수 = `R x C` 이고 각 칸에서 4방향 검사하므로 **O(R x C)**.
- **공간복잡도**: 최악의 경우 큐에 모든 정점이 동시에 들어갈 수 있어 **O(V)** (격자라면 O(R x C)).

## 흔한 실수 / 팁
- **방문 표시는 "큐에 넣을 때" 찍어라.** "큐에서 꺼낼 때" 찍으면, 같은 칸이 여러 번 큐에 들어가 시간이 폭발하고 거리도 틀어진다.
- **`pop()`은 값을 반환하지 않는다.** 값이 필요하면 반드시 `front()`로 먼저 읽고 나서 `pop()`. 이 둘을 빠뜨리면 무한 루프.
- **빈 큐에서 `front()`/`pop()` 호출 금지.** 미정의 동작(런타임 에러). 항상 `while(!q.empty())`로 보호.
- **레벨 단위 처리 시 `q.size()`를 루프 안에서 부르지 마라.** push 때문에 크기가 계속 변한다. 반드시 루프 시작 전에 `int sz = q.size();`로 고정.
- **시작점도 방문 처리 + 큐에 넣기.** 시작점의 거리는 0. 빼먹으면 시작점이 다시 탐색돼 꼬인다.
- **가중치가 칸마다 다르면 BFS로는 틀린다.** 그땐 다익스트라(우선순위 큐)나 0-1 BFS(덱)를 써야 한다.
- **자료형 주의.** 거리/카운트가 클 수 있으면 `int` 오버플로를 점검. 좌표는 `pair<int,int>`나 `1차원 인덱스(x*C+y)`로 다루면 깔끔하다.
- **큰 격자에서 `std::queue`가 느리게 느껴지면** 내부 컨테이너를 `deque`(기본)로 두되, 입출력은 `ios_base::sync_with_stdio(false)`로 가속하라. 정 급하면 직접 배열 큐(head/tail 인덱스)를 써도 된다.
- **방문 배열과 거리 배열을 합쳐라.** `dist[][]`를 -1로 초기화하면 `dist != -1`이 곧 "방문함"이라, 별도 `visited` 배열이 필요 없어 코드가 짧아진다.
