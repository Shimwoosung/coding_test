# 투 포인터 / 슬라이딩 윈도우

## 한 줄 요약
두 인덱스(lo, hi)를 같은 방향 또는 양 끝에서 움직여 O(N)에 구간/쌍을 찾는다.

## 언제 쓰나
- 정렬된 배열에서 합이 X인 쌍 찾기(양 끝 포인터)
- 연속 부분합 조건(슬라이딩 윈도우): 합이 S 이상인 최소 길이 등

## 왜 이거 + 대안 + 차이
- 모든 쌍/구간을 보면 O(N^2). 투포인터는 O(N) (정렬 포함 O(N log N)).
- 성립 조건: 한 포인터를 옮길 때 다른 포인터가 되돌아갈 필요가 없어야(단조성). 음수 섞이면 슬라이딩 윈도우 깨질 수 있음.

## 핵심
- 양 끝: 합<X면 lo++, 합>X면 hi--, 같으면 카운트.
- 윈도우: hi 확장하며 합 누적, 조건 넘으면 lo 축소.

## 흔한 실수
- 정렬 안 하고 양 끝 포인터 사용
- 합 변수 오버플로 → long long
