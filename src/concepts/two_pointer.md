# 투 포인터 (Two Pointer)

## 한 줄 요약
하나의 배열(또는 문자열)을 두 개의 인덱스(포인터)로 훑으면서, 두 포인터를 조건에 따라 좁히거나 넓혀서 **O(N²)로 풀 법한 문제를 O(N)에 푸는** 기법이다.

---

## 언제 쓰나
실제 코딩테스트에서 아래와 같은 신호가 보이면 투 포인터를 떠올려야 한다.

- **정렬된 배열에서 "합이 K가 되는 두 수"** 를 찾기 (`Two Sum II`)
- **연속된 부분 배열(구간)** 의 합/길이를 다루는 문제
  - "합이 S 이상인 가장 짧은 연속 부분 배열"
  - "합이 정확히 K인 부분 배열의 개수" (양수 배열일 때)
- **슬라이딩 윈도우** 류: "서로 다른 문자가 K개 이하인 가장 긴 부분 문자열"
- **두 개의 정렬된 배열 병합** (merge sort의 merge 단계)
- **회문(팰린드롬) 판별**: 양 끝에서 가운데로 좁혀오기
- **중복 제거**: 정렬된 배열에서 중복을 in-place로 제거
- "이 배열에서 조건을 만족하는 쌍/구간이 몇 개냐" 인데 N이 10^5 ~ 10^6이라 이중 for문(O(N²))이 시간 초과(TLE) 날 때

---

## 왜 이걸 쓰나 + 대안과 차이

핵심은 **불필요한 재탐색을 없애는 것**이다.

### 대안 1) 완전 탐색 (이중 for문)
"합이 K인 두 수"를 모든 쌍을 비교하면 O(N²). N=100,000이면 100억 번 연산 → TLE.
투 포인터는 정렬을 전제로 **양 끝에서 좁혀오면서** 각 원소를 사실상 한 번씩만 보므로 O(N).

### 대안 2) 해시맵 (Two Sum)
"합이 K인 두 수"는 해시맵으로도 O(N)에 풀린다. 다만:
- 해시맵: 정렬이 **필요 없음**, 하지만 추가 메모리 O(N), 상수 계수가 큼
- 투 포인터: **이미 정렬되어 있을 때** 추가 메모리 O(1)로 가장 깔끔

### 대안 3) 누적 합(Prefix Sum)
"구간 합"은 누적 합으로도 가능하지만, "조건을 만족하는 가장 긴/짧은 구간"처럼 **포인터를 점진적으로 움직여야 하는** 경우 투 포인터(슬라이딩 윈도우)가 더 직관적이고 빠르다.

### 한 줄 비교

| 방법 | 시간 | 추가 메모리 | 정렬 필요 |
|------|------|-------------|-----------|
| 이중 for문 | O(N²) | O(1) | X |
| 해시맵 | O(N) | O(N) | X |
| **투 포인터** | **O(N)** | **O(1)** | 보통 O (정렬됨 전제) |

**왜 투 포인터가 빠른가?**
정렬된 배열에서 "왼쪽 포인터를 오른쪽으로 옮기면 합은 절대 작아지지 않는다"는 **단조성(monotonicity)** 을 이용한다. 그래서 한 번 버린 후보는 다시 볼 필요가 없다. 이 "되돌아가지 않는다"가 O(N)의 비밀이다.

## 동작 과정

### 예제 A) 양 끝에서 좁히기 — 정렬된 배열에서 합이 `target = 9`인 두 수 찾기

배열: `[1, 2, 4, 5, 7, 11]` (오름차순 정렬됨)
포인터 `lo`는 맨 왼쪽, `hi`는 맨 오른쪽에서 시작.

```
규칙:
  sum = arr[lo] + arr[hi]
  sum == target  ->  찾았다! 종료
  sum <  target  ->  더 커야 하니 lo++  (왼쪽을 오른쪽으로)
  sum >  target  ->  더 작아야 하니 hi-- (오른쪽을 왼쪽으로)

index :   0   1   2   3   4   5
value : [ 1   2   4   5   7  11 ]
target = 9

[Step 1]  lo=0, hi=5
          1               11
          ^                ^
          lo               hi
          sum = 1 + 11 = 12  >  9   ->  hi--

[Step 2]  lo=0, hi=4
          1           7
          ^           ^
          lo          hi
          sum = 1 + 7 = 8   <  9   ->  lo++

[Step 3]  lo=1, hi=4
              2       7
              ^       ^
              lo      hi
          sum = 2 + 7 = 9   ==  9   ->  찾았다! (2, 7)
```

포인터가 서로를 향해 한 칸씩만 움직이므로, 두 포인터가 만날 때까지 **최대 N번** 반복 → O(N).

---

### 예제 B) 같은 방향(슬라이딩 윈도우) — 합이 `S = 8` 이상인 가장 짧은 연속 부분 배열

배열: `[2, 3, 1, 2, 4, 3]` (양수 배열, 정렬 X)
`start`(윈도우 왼쪽), `end`(윈도우 오른쪽)를 모두 0에서 시작.
`end`를 늘리며 합을 키우다가, 조건을 만족하면 `start`를 당겨서 길이를 줄인다.

```
[ ] 안이 현재 윈도우.  sum = 윈도우 안 합.  목표: sum >= 8

start=0
end=0 : [2] 3 1 2 4 3        sum=2   < 8  -> end 확장
end=1 : [2 3] 1 2 4 3        sum=5   < 8  -> end 확장
end=2 : [2 3 1] 2 4 3        sum=6   < 8  -> end 확장
end=3 : [2 3 1 2] 4 3        sum=8  >= 8  -> 길이4 기록, start 당김
        2 [3 1 2] 4 3        sum=6   < 8  -> end 확장
end=4 :  2 [3 1 2 4] 3       sum=10 >= 8  -> 길이4, start 당김
        2 3 [1 2 4] 3        sum=7   < 8  -> end 확장
end=5 :  2 3 [1 2 4 3]       sum=10 >= 8  -> 길이4... start 당김
        2 3 1 [2 4 3]        sum=9  >= 8  -> 길이3 기록! start 당김
        2 3 1 2 [4 3]        sum=7   < 8  -> end 확장 (끝)

정답: 가장 짧은 길이 = 3  (부분 배열 [4,3] 의 앞... 즉 [2,4,3] 중 [4,3]=7은 미만,
                            최소 길이를 만족시킨 구간은 길이 3)
```

`start`와 `end` 둘 다 한 방향으로만 이동하고 절대 뒤로 가지 않는다.
각 포인터가 최대 N번 전진 → 전체 O(2N) = **O(N)**.

## C++ 예제 코드

### (A) 정렬된 배열에서 합이 target인 두 수 — 양 끝 좁히기

```cpp
#include <bits/stdc++.h>
using namespace std;

// arr는 오름차순 정렬되어 있다고 가정.
// 합이 target인 두 인덱스를 (lo, hi)로 반환, 없으면 {-1,-1}.
pair<int,int> twoSumSorted(const vector<int>& arr, int target) {
    int lo = 0;                 // 왼쪽 포인터 (가장 작은 값 쪽)
    int hi = (int)arr.size()-1; // 오른쪽 포인터 (가장 큰 값 쪽)

    while (lo < hi) {           // 두 포인터가 교차하기 전까지
        int sum = arr[lo] + arr[hi];
        if (sum == target) {
            return {lo, hi};    // 정답 발견
        } else if (sum < target) {
            lo++;               // 합이 작다 -> 더 큰 값을 끌어와야 하니 왼쪽을 전진
        } else {                // sum > target
            hi--;               // 합이 크다 -> 더 작은 값을 끌어와야 하니 오른쪽을 후진
        }
    }
    return {-1, -1};            // 못 찾음
}

int main() {
    vector<int> arr = {1, 2, 4, 5, 7, 11};
    auto [i, j] = twoSumSorted(arr, 9);
    if (i != -1)
        cout << "found: arr[" << i << "]=" << arr[i]
             << " + arr[" << j << "]=" << arr[j] << "\n"; // 2 + 7
    else
        cout << "not found\n";
    return 0;
}
```

### (B) 슬라이딩 윈도우 — 합이 S 이상인 가장 짧은 연속 부분 배열 길이

```cpp
#include <bits/stdc++.h>
using namespace std;

// 양수로 이루어진 배열에서 합이 S 이상인 가장 짧은 연속 부분 배열의 길이.
// 그런 구간이 없으면 0을 반환.
int minSubArrayLen(int S, const vector<int>& arr) {
    int n = (int)arr.size();
    int start = 0;            // 윈도우 왼쪽 끝
    long long sum = 0;        // 현재 윈도우의 합
    int best = INT_MAX;       // 지금까지 찾은 최소 길이

    for (int end = 0; end < n; end++) {
        sum += arr[end];                 // 윈도우 오른쪽을 한 칸 넓힌다

        // 조건을 만족하는 동안 왼쪽을 최대한 당겨서 길이를 줄인다
        while (sum >= S) {
            best = min(best, end - start + 1); // 현재 윈도우 길이 갱신
            sum -= arr[start];                 // 왼쪽 원소를 윈도우에서 제거
            start++;                           // 왼쪽 포인터 전진
        }
    }
    return (best == INT_MAX) ? 0 : best;
}

int main() {
    vector<int> arr = {2, 3, 1, 2, 4, 3};
    cout << minSubArrayLen(8, arr) << "\n"; // 2  ([4,3] 은 7이라 안 되고, 최소는 길이 2의 ... )
    // 참고: 위 배열에서 합>=8인 최단 구간은 [4,3]=7(X), 실제 최단은 길이 2 미존재 -> 길이 3.
    return 0;
}
```

> 참고: 슬라이딩 윈도우(같은 방향 투 포인터)는 **배열 원소가 모두 양수일 때** 단조성이 보장된다. 음수가 섞이면 "오른쪽을 넓힐수록 합이 커진다"가 깨지므로 누적 합 + 해시맵 등 다른 기법을 써야 한다.

### (C) 회문 판별 — 양 끝에서 가운데로

```cpp
#include <bits/stdc++.h>
using namespace std;

bool isPalindrome(const string& s) {
    int l = 0, r = (int)s.size() - 1;
    while (l < r) {
        if (s[l] != s[r]) return false; // 양 끝 문자가 다르면 회문 아님
        l++;                            // 왼쪽 전진
        r--;                            // 오른쪽 후진
    }
    return true;
}

int main() {
    cout << boolalpha << isPalindrome("level") << "\n"; // true
    cout << boolalpha << isPalindrome("hello") << "\n"; // false
    return 0;
}
```

## 시간복잡도

| 상황 | 시간복잡도 | 비고 |
|------|-----------|------|
| 정렬이 이미 되어 있을 때 | **O(N)** | 각 포인터가 최대 N번 이동 |
| 정렬을 직접 해야 할 때 | **O(N log N)** | 정렬이 지배항 |
| 공간복잡도 | **O(1)** | 포인터 변수만 추가로 사용 (in-place) |

- 양 끝 좁히기: `lo`는 오른쪽으로, `hi`는 왼쪽으로만 이동 → 둘이 만날 때까지 총 N번.
- 슬라이딩 윈도우: `start`와 `end`가 각각 0→N으로 한 번씩만 전진 → 총 2N번 = O(N).
- 핵심은 **포인터가 절대 뒤로 되돌아가지 않는다**는 점. 이중 반복문처럼 보여도 실제 총 이동 횟수가 선형이다.

## 흔한 실수 / 팁

1. **정렬을 안 했다.**
   양 끝 좁히기 투 포인터는 **정렬된 배열**이 대전제다. 정렬 안 된 배열에 쓰면 틀린다. 입력이 정렬되어 있는지 먼저 확인하고, 아니면 `sort()` 후 적용하라. (단, 인덱스 원본이 필요하면 정렬 전 인덱스를 따로 보관.)

2. **루프 조건 `lo < hi` vs `lo <= hi`.**
   서로 다른 두 원소를 찾을 때는 `lo < hi`. 같은 원소를 두 번 써도 되는 문제(예: 가운데 한 원소)라면 `lo <= hi`. 문제 조건을 정확히 읽어라.

3. **슬라이딩 윈도우에 음수가 섞인 배열을 쓴다.**
   같은 방향 투 포인터는 "넓히면 커지고, 좁히면 작아진다"는 단조성이 필요하다. 음수가 있으면 깨진다. 이때는 누적 합 + 해시맵으로 전환.

4. **합 변수 오버플로.**
   N이 크고 값이 크면 `int` 합이 오버플로된다. `long long`을 쓰자. (위 예제 B처럼)

5. **윈도우 갱신 순서 헷갈림.**
   슬라이딩 윈도우에서 "더한 다음 줄인다" 순서를 지켜라. `end`를 더해 윈도우를 넓힌 **직후**에 조건을 검사하고 `start`를 당긴다. 더하기 전에 검사하면 빈 윈도우/off-by-one 오류가 난다.

6. **중복 처리.**
   "합이 K인 쌍의 개수"처럼 중복을 세거나 건너뛰어야 할 때, 같은 값이 연속되면 `while`로 같은 값을 스킵하는 처리를 추가해야 중복 카운트를 막을 수 있다.

7. **정답 길이 초기값.**
   최소 길이를 구할 땐 `INT_MAX`로 초기화하고 끝에 "갱신 안 됐으면 0/없음" 처리. 최대 길이를 구할 땐 `0`으로 초기화.

8. **언제 떠올릴까 — 신호 정리.**
   "정렬된 배열 + 쌍/합", "연속 구간의 최대/최소 길이", "N이 커서 O(N²)가 안 됨" 이 세 가지 중 하나라도 보이면 투 포인터를 가장 먼저 의심하라.
