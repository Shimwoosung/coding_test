# 비트마스크 (Bitmask)

## 한 줄 요약
정수의 각 **비트(bit)** 하나하나를 "켜짐(1) / 꺼짐(0)" 플래그로 사용하여, 여러 개의 boolean 상태나 "부분집합(subset)"을 **하나의 정수**로 압축해서 빠르고 메모리 효율적으로 다루는 기법이다.

---

## 언제 쓰나
- **"N개 중 일부를 선택하는 모든 경우"를 전부 탐색**해야 할 때 (N이 작음, 보통 N ≤ 20).
  - 예: "물건 N개 중 일부를 골라 무게 합이 K가 되는 경우의 수"
- **방문 상태를 정수 한 개로 표현**해야 하는 DP.
  - 예: **외판원 순회 문제(TSP)** — "어떤 도시들을 이미 방문했는가"를 비트마스크로 표현.
  - 예: **집합을 상태로 쓰는 비트 DP** — "지금까지 사용한 알파벳 집합", "켜진 전구 집합" 등.
- **집합 연산(합집합/교집합/차집합/포함 여부)을 O(1)에 빠르게** 하고 싶을 때.
- **여러 개의 on/off 옵션을 함수 인자 하나로** 넘기고 싶을 때 (권한 플래그 등).
- 격자/보드 문제에서 **한 줄의 상태(어디에 말을 놓았나)** 를 정수로 압축할 때 (예: 벽돌/조명/줄세우기 DP).

핵심 신호: **N ≤ 20 정도로 작고, "부분집합 전체"를 다뤄야 한다** → 비트마스크를 떠올려라.

---

## 왜 이걸 쓰나 + 대안과 차이

| 방법 | 부분집합 표현 | 집합 연산 속도 | 메모리 | 비고 |
|------|--------------|---------------|--------|------|
| `bool visited[N]` 배열 | 배열 1개 | 원소 단위 O(N) | N바이트 | 직관적이지만 "상태"를 통째로 키로 쓰기 어려움 |
| `std::set` / `std::vector` | 컨테이너 | 느림(동적할당) | 큼 | DP 상태 키로 부적합 |
| **비트마스크 (정수)** | **정수 1개** | **O(1)** | **4~8바이트** | 배열 인덱스로 바로 사용 가능 |
| `std::bitset` | 고정 크기 비트셋 | O(N/word) | 작음 | N이 클 때(수백~수천) 유리, 인덱싱은 편함 |

### 핵심 차이
- **상태를 배열 인덱스로 쓸 수 있다.** `bool[N]` 배열은 `dp[배열]` 같은 게 안 되지만, 비트마스크 `mask`는 정수라서 `dp[mask]`로 곧장 메모이제이션 테이블의 인덱스가 된다. 이게 **비트 DP**의 핵심 무기다.
- **집합 연산이 한 줄.** "두 집합 합치기"가 `a | b`, "공통 원소 있나"가 `a & b`, 이런 식으로 CPU 명령 한 방에 끝난다.
- **N이 작아야 한다.** 부분집합 개수는 $2^N$. N=20이면 약 100만, N=25면 3300만으로 슬슬 버겁다. **N ≤ 20~22가 현실적 한계** (메모리/시간 모두). 그 이상이면 비트마스크 대신 다른 접근(그리디, 흐름, 분할정복 등)을 고민해야 한다.

---

## 동작 과정

정수 `mask`의 비트를 "오른쪽이 0번 자리"로 읽는다. 예를 들어 원소가 5개(0~4번)인 집합에서 `{0, 2, 3}`을 선택한 상태는 다음과 같다.

```
비트 위치 (자리값):   4    3    2    1    0
                      16    8    4    2    1
                    +----+----+----+----+----+
선택 여부 (mask):   |  0 |  1 |  1 |  0 |  1 |   <- 3번,2번,0번이 켜짐
                    +----+----+----+----+----+
                       원소4 원소3 원소2 원소1 원소0

정수 값 = 8 + 4 + 1 = 13  (이진수로 01101)
즉  mask = 13  ==  집합 {0, 2, 3}
```

### 1) 특정 비트 켜기 / 끄기 / 확인 / 토글

```
기준 mask = 01101 (=13, 집합 {0,2,3})

[i번 비트 검사]  mask & (1<<i)
  i=2: (1<<2)=00100 ,  01101 & 00100 = 00100 != 0  ->  2번은 있음(켜짐)
  i=1: (1<<1)=00010 ,  01101 & 00010 = 00000 == 0  ->  1번은 없음(꺼짐)

[i번 비트 켜기]  mask | (1<<i)        // 1번 추가
  01101 | 00010 = 01111 (=15)  ->  {0,1,2,3}
        ^ 여기 1로 바뀜

[i번 비트 끄기]  mask & ~(1<<i)       // 2번 제거
  ~(1<<2) = 11011
  01101 & 11011 = 01001 (=9)   ->  {0,3}
       ^ 여기 0으로 바뀜

[i번 비트 토글]  mask ^ (1<<i)        // 0번 뒤집기
  01101 ^ 00001 = 01100 (=12)  ->  {2,3}
            ^ 1이었으니 0으로
```

### 2) 모든 부분집합 순회 (0 ~ 2^N - 1)

N=3일 때 `for (mask = 0; mask < (1<<3); mask++)` 가 도는 모습:

```
mask=000 (0)  ->  { }            (공집합)
mask=001 (1)  ->  {0}
mask=010 (2)  ->  {1}
mask=011 (3)  ->  {0,1}
mask=100 (4)  ->  {2}
mask=101 (5)  ->  {0,2}
mask=110 (6)  ->  {1,2}
mask=111 (7)  ->  {0,1,2}        (전체집합)

총 2^3 = 8가지. mask를 0부터 7까지 올리기만 하면
"가능한 모든 선택"을 한 번씩 빠짐없이 방문한다.
```

### 3) 집합 연산 한눈에

```
 A = {0,1,3}  -> 01011
 B = {1,2}    -> 00110

 합집합  A|B = 01111  -> {0,1,2,3}
 교집합  A&B = 00010  -> {1}
 차집합  A & ~B
        ~B = 11001
        01011 & 11001 = 01001 -> {0,3}   (A에서 B를 뺌)
 대칭차  A^B = 01101  -> {0,2,3}          (한쪽에만 있는 원소)
 부분집합? (A가 B를 포함?)  (A & B) == B
```

## C++ 예제 코드

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int N = 5; // 원소 0..4

    // ── (1) 기본 비트 연산 ─────────────────────────────
    int mask = 0;                 // 공집합에서 시작 (00000)
    mask |= (1 << 0);             // 0번 추가  -> 00001
    mask |= (1 << 2);             // 2번 추가  -> 00101
    mask |= (1 << 3);             // 3번 추가  -> 01101  (= 13)

    // i번 원소가 집합에 있는지 검사
    for (int i = 0; i < N; i++) {
        if (mask & (1 << i))      // 0이 아니면 켜져 있음
            cout << i << " ";     // 출력: 0 2 3
    }
    cout << "\n";

    mask &= ~(1 << 2);            // 2번 제거  -> 01001 (= 9)
    mask ^= (1 << 1);             // 1번 토글  -> 01011 (= 11)

    // 켜진 비트 개수(=집합 크기). GCC/Clang 내장 함수
    cout << "size = " << __builtin_popcount(mask) << "\n";

    // ── (2) 모든 부분집합 완전탐색 ─────────────────────
    // "원소 0..N-1 중 일부를 골랐을 때 값의 합이 최대가 되는 경우"의 틀
    int val[5] = {3, 1, 4, 1, 5};
    int best = 0;
    for (int s = 0; s < (1 << N); s++) {   // s: 0 ~ 2^N-1 모든 부분집합
        int sum = 0;
        for (int i = 0; i < N; i++)
            if (s & (1 << i))               // i번이 선택됐다면
                sum += val[i];
        best = max(best, sum);
    }
    cout << "max subset sum = " << best << "\n"; // 14 (전부 선택)

    // ── (3) 비트마스크 DP: 외판원 순회(TSP) 골격 ──────
    // dp[visited][last] = visited 도시들을 방문했고 현재 last에 있을 때 최소비용
    int n = 4;
    int dist[4][4] = {
        {0, 10, 15, 20},
        {10, 0, 35, 25},
        {15, 35, 0, 30},
        {20, 25, 30, 0}
    };
    const int INF = 1e9;
    vector<vector<int>> dp(1 << n, vector<int>(n, INF));
    dp[1][0] = 0;                 // 0번 도시에서 출발(0번만 방문, 비트 0 켜짐)

    for (int visited = 0; visited < (1 << n); visited++) {
        for (int last = 0; last < n; last++) {
            if (dp[visited][last] == INF) continue;      // 도달 불가 상태 skip
            if (!(visited & (1 << last))) continue;       // last가 방문집합에 없으면 무효
            for (int nxt = 0; nxt < n; nxt++) {
                if (visited & (1 << nxt)) continue;       // 이미 방문한 도시는 패스
                int nv = visited | (1 << nxt);            // nxt를 방문 처리
                dp[nv][nxt] = min(dp[nv][nxt],
                                  dp[visited][last] + dist[last][nxt]);
            }
        }
    }
    // 모든 도시 방문 후 0번으로 복귀하는 최소 비용
    int full = (1 << n) - 1;      // 모든 비트가 1 (전체 방문)
    int ans = INF;
    for (int last = 0; last < n; last++)
        ans = min(ans, dp[full][last] + dist[last][0]);
    cout << "TSP min cost = " << ans << "\n"; // 80

    return 0;
}
```

### 자주 쓰는 관용구 모음
```cpp
(1 << i)                 // i번 비트만 켜진 값
mask & (1 << i)          // i번 비트가 켜졌나? (0/비0)
mask | (1 << i)          // i번 비트 켜기
mask & ~(1 << i)         // i번 비트 끄기
mask ^ (1 << i)          // i번 비트 토글
(1 << N) - 1             // 하위 N개 비트가 모두 1 (전체집합)
__builtin_popcount(x)    // 켜진 비트 수 (int용, long long은 ...ll)
__builtin_ctz(x)         // 가장 낮은 켜진 비트의 위치 (x != 0일 때)
mask & (mask - 1)        // 가장 낮은 켜진 비트 하나 제거
mask & (-mask)           // 가장 낮은 켜진 비트만 추출

// mask의 모든 "부분집합"만 빠르게 순회하는 고급 관용구
for (int sub = mask; sub > 0; sub = (sub - 1) & mask) {
    // sub는 mask에 포함된 부분집합들만 내려간다
}
```

## 시간복잡도
- **부분집합 전체 완전탐색**: 부분집합 $2^N$개 × 각 비트 확인 $O(N)$ = **$O(N \cdot 2^N)$**.
  - N=20 → 약 $20 \times 10^6 = 2 \times 10^7$ (충분히 가능).
- **비트마스크 DP (TSP류)**: 상태 $2^N \times N$개, 각 상태에서 전이 $O(N)$ = **$O(N^2 \cdot 2^N)$**.
  - N=16 → 약 $16^2 \times 65536 \approx 1.7 \times 10^7$ (가능). N=20쯤이 사실상 상한.
- **개별 비트 연산** (켜기/끄기/검사): 각각 **$O(1)$**.
- **메모리**: `dp[1<<N][N]` 같은 테이블은 $O(N \cdot 2^N)$. N=20에서 int 기준 약 80MB 이상 → 메모리 제한 주의.

---

## 흔한 실수 / 팁
- **`1 << i`의 오버플로우.** `i >= 31`이면 `int` 범위를 넘는다. N이 31 이상일 가능성이 있으면 반드시 **`1LL << i`** (long long)로 써라. 비트 31에서 부호 비트를 건드려 음수가 되는 사고가 흔하다.
- **연산자 우선순위.** `&`, `|`, `^`는 비교 연산자(`==`, `!=`)보다 **우선순위가 낮다**. `if (mask & (1<<i) == 0)`은 `1<<i == 0`이 먼저 계산되어 버그가 된다. **`if ((mask & (1<<i)) == 0)`** 처럼 괄호를 꼭 쳐라.
- **"켜졌는지" 검사는 `!= 0`로.** `mask & (1<<i)`의 결과는 1이 아니라 $2^i$일 수 있다. `== 1`로 비교하면 i=0일 때만 맞는다. 진짜/거짓 판정만 하거나 `!= 0`을 써라.
- **반복 범위는 `(1 << N)`까지.** `for (mask = 0; mask <= (1<<N); mask++)`처럼 `<=`를 쓰면 마지막에 한 번 더 돌며 인덱스 초과(`dp[1<<N]`)가 난다. **`< (1 << N)`** 이 맞다.
- **전체집합은 `(1<<N) - 1`.** `1<<N`이 아니다. N개 비트가 모두 1인 값은 한 칸 적은 `(1<<N)-1`.
- **`__builtin_popcount`는 `int`용.** `long long` 마스크엔 `__builtin_popcountll`을 써야 한다. 안 그러면 상위 32비트를 못 센다.
- **N이 작을 때만 쓴다.** N=30짜리에 $2^N$ 루프를 돌리면 10억이 넘어 시간 초과. 문제의 N 제약을 먼저 확인하고 비트마스크가 적합한지 판단하라(보통 N ≤ 20~22).
- **DP 전이 방향 주의.** 비트 DP는 보통 `visited`가 커지는 방향(원소가 추가되는 방향)으로 채워야 한다. `mask`를 0부터 증가시키며 순회하면 자연스럽게 "더 적게 방문한 상태 → 더 많이 방문한 상태" 순서가 보장된다.
