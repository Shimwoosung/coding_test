# 그리디 (Greedy)

## 한 줄 요약
매 순간 "지금 가장 좋아 보이는 선택"을 하는 전략. 단, 그게 전체 최적임을 보장할 때만 옳다.

## 언제 쓰나
- 동전 0 (배수 관계 동전 → 큰 것부터)
- 회의실 배정 (끝나는 시간 빠른 것부터)
- ATM (짧은 작업부터), 카드 정렬 (작은 두 묶음부터 = 최소 힙)

## 왜 이거 + 대안 + 차이
- DP는 모든 경우를 따져 최적을 보장. 그리디는 빠르지만(정렬 O(N log N)) "탐욕적 선택이 최적"이라는 증명이 필요.
- 동전이 배수 관계가 아니면 그리디가 틀림 → 그땐 DP(동전2).

## 핵심 패턴
- 보통 "무엇을 기준으로 정렬하느냐"가 전부.
- 우선순위 큐(최소/최대 힙)로 매번 최적 원소를 꺼내는 형태도 그리디.

## 흔한 실수
- 정렬 기준을 잘못 잡음 (회의실은 종료시간 기준이지 시작시간 X)
- 그리디가 성립하지 않는 문제에 그리디 적용
