# 동적 계획법 (Dynamic Programming, DP)

## 한 줄 요약
**큰 문제를 작은 문제로 나누고, 작은 문제의 답을 한 번만 계산해서 저장(메모)해 두고 재사용**함으로써, 같은 계산을 반복하지 않게 만드는 기법이다.

---

## 언제 쓰나
DP는 다음 두 성질을 동시에 가진 문제에서 강력하다.

1. **최적 부분 구조 (Optimal Substructure)**: 큰 문제의 최적해가 작은 문제들의 최적해로 조립된다.
2. **중복되는 부분 문제 (Overlapping Subproblems)**: 같은 작은 문제가 여러 번 반복해서 등장한다.

실제 코딩테스트 문제 유형 예시:

- **피보나치 수 / 계단 오르기**: `f(n) = f(n-1) + f(n-2)` 처럼 점화식이 자기 자신을 부르는 문제
- **배낭 문제 (Knapsack)**: 무게 한도 안에서 가치 최대화
- **최장 증가 부분 수열 (LIS)**: 배열에서 증가하는 가장 긴 수열 길이
- **편집 거리 (Edit Distance)**: 문자열 A를 B로 바꾸는 최소 연산 횟수
- **동전 교환 (Coin Change)**: 금액을 만드는 최소 동전 개수 / 경우의 수
- **격자 경로 수**: 좌상단에서 우하단까지 가는 경로의 수 (장애물 포함)
- **구간 DP / 행렬 곱셈 순서**: 구간을 나눠 합치는 비용 최소화

핵심 신호: "최솟값/최댓값/경우의 수를 구하라"인데, **선택의 단계마다 이전 결정 결과가 다음에 영향을 주고**, 완전 탐색하면 같은 상태를 수없이 반복 계산하게 될 때.

---

## 왜 이걸 쓰나 + 대안과 차이

| 기법 | 핵심 아이디어 | DP와의 차이 |
|------|--------------|-------------|
| **완전 탐색 (브루트포스)** | 모든 경우를 다 해본다 | 중복 계산을 막지 못해 지수 시간(`O(2^n)`)으로 폭발 |
| **분할 정복 (Divide & Conquer)** | 큰 문제를 작은 문제로 나눔 | 나눈 문제들이 **서로 겹치지 않음** (예: 병합 정렬). DP는 **겹치는** 문제를 메모로 재사용 |
| **그리디 (Greedy)** | 매 순간 가장 좋아 보이는 선택 | 한 번 고른 건 되돌아보지 않음. 빠르지만 **항상 최적은 아님**. DP는 모든 경우를 따져 최적 보장 |
| **DP** | 작은 문제 답을 저장하고 재사용 | 시간을 메모리와 맞바꿔(trade-off) 중복 제거. 다항 시간으로 끌어내림 |

**왜 완전 탐색이 망하는가 — 피보나치 예시:**

```
재귀만으로 f(5)를 계산하면 같은 값을 몇 번이나 다시 구할까?

                    f(5)
                  /      \
              f(4)        f(3)
             /    \      /    \
          f(3)   f(2)  f(2)  f(1)
         /   \   /  \   /  \
      f(2) f(1)f(1)f(0)f(1)f(0)
      / \
   f(1) f(0)

f(3)을 2번, f(2)를 3번 중복 계산!  n이 커지면 O(2^n)으로 폭발한다.
DP는 f(2), f(3) ... 을 "한 번만" 계산해 표에 적어두고 꺼내 쓴다.
```

**DP를 구현하는 두 가지 방식:**

- **Top-Down (메모이제이션)**: 재귀로 큰 문제부터 부르되, 이미 푼 문제는 캐시에서 꺼냄. 코드가 점화식과 닮아 직관적.
- **Bottom-Up (타뷸레이션)**: 가장 작은 문제부터 표를 채워 올라감. 재귀 오버헤드/스택 오버플로가 없어 보통 더 빠름.

## 동작 과정

### 예시 1: 피보나치 — Bottom-Up 표 채우기

```
점화식:  dp[i] = dp[i-1] + dp[i-2]
초기값:  dp[0] = 0,  dp[1] = 1

인덱스:   0    1    2    3    4    5    6
        +----+----+----+----+----+----+----+
초기:   | 0  | 1  | ?  | ?  | ?  | ?  | ?  |
        +----+----+----+----+----+----+----+

i=2:  dp[2] = dp[1] + dp[0] = 1 + 0 = 1
        +----+----+----+----+----+----+----+
        | 0  | 1  | 1  | ?  | ?  | ?  | ?  |
        +----+----+----+----+----+----+----+
                     ^---- 새로 채움

i=3:  dp[3] = dp[2] + dp[1] = 1 + 1 = 2
        +----+----+----+----+----+----+----+
        | 0  | 1  | 1  | 2  | ?  | ?  | ?  |
        +----+----+----+----+----+----+----+
                          ^

i=4:  dp[4] = dp[3] + dp[2] = 2 + 1 = 3
        +----+----+----+----+----+----+----+
        | 0  | 1  | 1  | 2  | 3  | ?  | ?  |
        +----+----+----+----+----+----+----+
                               ^

i=5:  dp[5] = dp[4] + dp[3] = 3 + 2 = 5
        +----+----+----+----+----+----+----+
        | 0  | 1  | 1  | 2  | 3  | 5  | ?  |
        +----+----+----+----+----+----+----+
                                    ^

i=6:  dp[6] = dp[5] + dp[4] = 5 + 3 = 8
        +----+----+----+----+----+----+----+
        | 0  | 1  | 1  | 2  | 3  | 5  | 8  |
        +----+----+----+----+----+----+----+
                                         ^
완성!  각 칸은 "딱 한 번"만 계산된다. -> O(n)
```

### 예시 2: 0/1 배낭 문제 — 2차원 표 채우기

```
물건: (무게 w, 가치 v)
  물건1: (1, 15)
  물건2: (3, 20)
  물건3: (4, 30)
배낭 용량 W = 4

dp[i][c] = "물건 1..i 까지 고려, 용량 c 일 때 얻을 수 있는 최대 가치"

점화식:
  넣지 않으면:  dp[i-1][c]
  넣으면(c >= w[i]):  dp[i-1][c - w[i]] + v[i]
  dp[i][c] = max( 위 둘 )

       용량 c -->   0    1    2    3    4
                 +----+----+----+----+----+
물건0(없음)      | 0  | 0  | 0  | 0  | 0  |
                 +----+----+----+----+----+
물건1 (1,15)     | 0  | 15 | 15 | 15 | 15 |   c>=1이면 15 담을 수 있음
                 +----+----+----+----+----+
물건2 (3,20)     | 0  | 15 | 15 | 20 | 35 |   c=4: max(15, dp[1][4-3]+20=15+20=35)=35
                 +----+----+----+----+----+
물건3 (4,30)     | 0  | 15 | 15 | 20 | 35 |   c=4: max(35, dp[2][4-4]+30=0+30=30)=35
                 +----+----+----+----+----+
                                        ^
                                정답 = dp[3][4] = 35
                          (물건1 + 물건2 선택: 무게 1+3=4, 가치 15+20=35)
```

## C++ 예제 코드

```cpp
#include <bits/stdc++.h>
using namespace std;

// ---------------------------------------------------------
// (1) 피보나치 - Top-Down (메모이제이션)
// ---------------------------------------------------------
long long memo[100];        // -1로 초기화: 아직 계산 안 함을 의미
long long fibTopDown(int n) {
    if (n <= 1) return n;               // 기저 조건(base case)
    if (memo[n] != -1) return memo[n];  // 이미 계산했으면 즉시 반환(중복 제거)
    // 처음 계산하는 경우에만 재귀 호출하고 결과를 저장
    return memo[n] = fibTopDown(n - 1) + fibTopDown(n - 2);
}

// ---------------------------------------------------------
// (2) 피보나치 - Bottom-Up (타뷸레이션)
// ---------------------------------------------------------
long long fibBottomUp(int n) {
    if (n <= 1) return n;
    vector<long long> dp(n + 1);
    dp[0] = 0; dp[1] = 1;               // 가장 작은 문제부터 채운다
    for (int i = 2; i <= n; i++)
        dp[i] = dp[i - 1] + dp[i - 2];  // 점화식 그대로
    return dp[n];
    // 참고: 직전 두 값만 필요하므로 변수 2개로 O(1) 공간 최적화도 가능
}

// ---------------------------------------------------------
// (3) 0/1 배낭 문제 - Bottom-Up
//   weight[i], value[i] : i번째 물건의 무게/가치 (1-indexed)
//   W : 배낭 최대 용량
// ---------------------------------------------------------
int knapsack(vector<int>& weight, vector<int>& value, int W) {
    int n = weight.size() - 1;          // 물건 개수 (1..n 사용)
    // dp[i][c] = 물건 1..i 고려, 용량 c 일 때 최대 가치
    vector<vector<int>> dp(n + 1, vector<int>(W + 1, 0));

    for (int i = 1; i <= n; i++) {
        for (int c = 0; c <= W; c++) {
            // 1) i번 물건을 넣지 않는 경우
            dp[i][c] = dp[i - 1][c];
            // 2) 용량이 충분하면 넣는 경우와 비교
            if (c >= weight[i])
                dp[i][c] = max(dp[i][c],
                               dp[i - 1][c - weight[i]] + value[i]);
        }
    }
    return dp[n][W];
}

int main() {
    // (1)(2) 피보나치 테스트
    memset(memo, -1, sizeof(memo));
    cout << "fib(10) top-down  = " << fibTopDown(10) << "\n";   // 55
    cout << "fib(10) bottom-up = " << fibBottomUp(10) << "\n";  // 55

    // (3) 배낭 테스트 (1-indexed: 0번 칸은 더미)
    vector<int> w = {0, 1, 3, 4};   // 물건1=1, 물건2=3, 물건3=4
    vector<int> v = {0, 15, 20, 30};
    cout << "knapsack(W=4)     = " << knapsack(w, v, 4) << "\n"; // 35
    return 0;
}
```

## 시간복잡도

DP의 시간복잡도는 보통 **(상태의 개수) x (한 상태를 채우는 비용)** 으로 계산한다.

| 문제 | 상태 개수 | 한 상태 비용 | 전체 시간 | 공간 |
|------|-----------|-------------|-----------|------|
| 피보나치 | `O(n)` | `O(1)` | **`O(n)`** | `O(n)` → `O(1)` 최적화 가능 |
| 0/1 배낭 | `O(n·W)` | `O(1)` | **`O(n·W)`** | `O(n·W)` → `O(W)` (1차원) 가능 |
| LIS (DP) | `O(n)` | `O(n)` | **`O(n²)`** (이분탐색 시 `O(n log n)`) | `O(n)` |
| 편집 거리 | `O(n·m)` | `O(1)` | **`O(n·m)`** | `O(n·m)` → `O(m)` 가능 |

> 완전 탐색의 `O(2^n)` 같은 지수 시간을, 상태를 한 번씩만 푸는 다항 시간으로 끌어내리는 것이 DP의 핵심 가치다.

## 흔한 실수 / 팁

- **점화식과 기저 조건을 먼저 종이에 쓴다.** `dp[i]`가 "정확히 무엇을 의미하는지" 한 문장으로 정의하지 못하면 코드도 틀린다. (예: "dp[i] = i번째까지 봤을 때의 최댓값")
- **메모이제이션 초기화 값에 주의.** `0`이 유효한 답일 수 있는 문제에서 캐시 초기값을 `0`으로 두면 "계산 안 됨"과 구분이 안 된다. `-1`이나 별도 방문 배열을 쓰자.
- **Bottom-Up은 채우는 순서가 생명.** `dp[i]`를 계산할 때 필요한 `dp[i-1]`, `dp[i-2]`가 **이미 채워져 있어야** 한다. 루프 방향(작은 것 → 큰 것)을 항상 확인.
- **1차원 배낭 공간 최적화 시 순회 방향 주의.** 0/1 배낭을 1차원으로 줄이면 용량 `c`를 **큰 쪽에서 작은 쪽으로(역순)** 돌아야 같은 물건을 두 번 담는 버그를 막는다. (무한히 담는 "무한 배낭"은 정순)
- **오버플로 조심.** 경우의 수/큰 피보나치는 `int` 범위를 금방 넘는다. `long long`을 쓰거나 문제에서 요구하는 `MOD`로 나눠 저장하자.
- **Top-Down 깊은 재귀는 스택 오버플로 위험.** `n`이 매우 크면 Bottom-Up이 안전하다.
- **인덱스 오프셋 실수.** 1-indexed로 표를 잡으면 `dp[0]`(더미)와 경계 조건이 깔끔해지는 경우가 많다. 0-indexed와 헷갈리지 말 것.
- **"이 문제가 DP인가?" 판별법.** 완전 탐색 그림을 그렸을 때 **같은 부분 문제가 두 번 이상 나오면** DP로 캐싱할 여지가 있다.
