# 동적 계획법 (Dynamic Programming)

## 한 줄 요약
큰 문제를 작은 부분 문제로 나누고, 그 답을 저장(메모)해 재사용한다.

## 언제 쓰나
- 1로 만들기, 계단 오르기 (이전 상태로부터 점화식)
- 배낭(Knapsack), RGB거리, LIS — "선택의 누적 최적"

## 왜 이거 + 대안 + 차이
- 완전탐색은 같은 부분 문제를 중복 계산 → 지수 시간. DP는 한 번만 계산해 O(상태수).
- 그리디로 안 되는 "전역 최적"을 보장. 단, 상태 정의와 점화식이 핵심.

## 두 가지 방식
- Top-down (재귀 + 메모이제이션): dp[n] 직접 호출, 캐시.
- Bottom-up (반복문 + 테이블): dp[0]부터 채워 올라감. (대체로 빠르고 안전)

## 핵심
- "dp[i]가 무엇을 의미하는가"를 먼저 정의 → 점화식 → 초기값 → 정답 위치.
- 배낭은 무게를 거꾸로 도는 이유(0/1) 꼭 이해.

## 흔한 실수
- 상태 정의 모호 → 점화식이 안 나옴
- 초기값/경계 처리 누락
