# 정렬 (Sorting)

## 한 줄 요약

정렬은 데이터를 일정한 기준(오름차순/내림차순)에 따라 순서대로 재배열하는 것이며, 정렬된 데이터는 탐색·중복 제거·그리디·투포인터 등 수많은 알고리즘의 전제 조건이 되는 가장 기본적인 도구다.

## 언제 쓰나

- 출력 자체를 "크기 순서대로" 내야 할 때 (성적순, 좌표순, 사전순)
- 정렬 후 **이분 탐색(binary search)**으로 빠르게 값을 찾고 싶을 때
- 정렬 후 **인접한 원소만 비교**하면 중복 제거·최소 간격 찾기가 쉬워질 때
- **그리디(greedy)** 문제에서 "가장 작은/큰 것부터" 처리해야 할 때 (회의실 배정, 동전, 작업 스케줄링)
- **투 포인터 / 슬라이딩 윈도우**의 전제로 정렬이 필요할 때
- 여러 기준으로 줄을 세워야 할 때 (1순위 점수 내림차순, 동점이면 이름 오름차순)
- 좌표 압축(coordinate compression), 구간 스위핑(sweep line)의 전처리 단계

## 왜 이걸 쓰나 + 대안과 차이

정렬은 "무질서한 데이터에 질서를 부여"하는 행위다. 한 번 질서가 생기면 그 뒤의 작업이 극적으로 쉬워진다.

| 상황 | 정렬 없이 | 정렬 후 |
|------|-----------|---------|
| 특정 값 찾기 | 선형 탐색 O(N) | 이분 탐색 O(log N) |
| 중복 제거 | 해시/이중반복 | 인접 비교 O(N) |
| 가장 가까운 두 수 | 모든 쌍 O(N²) | 인접 차이 O(N) |
| 그리디 선택 | 매번 최솟값 탐색 | 앞에서부터 순회 |

**다른 도구와의 비교**

- **해시(unordered_map/set)**: 순서가 필요 없고 "존재 여부/카운트"만 빠르게(평균 O(1)) 알고 싶다면 해시가 낫다. 하지만 "순서"가 필요하면 정렬이 답이다.
- **우선순위 큐(priority_queue)**: 데이터가 계속 들어오고 나가는 **스트리밍** 상황에서 매번 최소/최대만 꺼내면 된다면 힙이 적합하다. 전체를 한 번에 줄 세울 거라면 정렬이 간단하다.
- **카운팅/기수 정렬**: 값의 범위가 작은 정수라면 비교 정렬의 O(N log N) 하한을 넘어 O(N)으로도 가능하다.
- **직접 구현(버블/선택/삽입) vs 표준 라이브러리**: 코딩테스트에서는 거의 항상 `std::sort`(인트로소트, 평균 O(N log N))를 쓰는 게 정답이다. 직접 정렬을 짜는 건 O(N²)이라 N이 조금만 커져도 시간 초과가 난다. 정렬 알고리즘의 "원리"는 이해하되, 실전에서는 라이브러리를 쓰자.

비교 정렬(merge/quick/heap 등)은 원소를 둘씩 비교해서 순서를 정하기 때문에 이론적으로 **O(N log N)** 보다 빠를 수 없다(비교 정렬의 하한).

## 동작 과정

### 1) 병합 정렬 (Merge Sort) — "나누고 합치기"

배열을 절반씩 끝까지 쪼갠 뒤, 정렬된 두 조각을 다시 합치면서 전체를 정렬한다. 항상 O(N log N)이고 안정 정렬이다.

```
초기 배열:        [ 5  2  8  1  9  3 ]

[분할 단계 — 계속 반으로 쪼갠다]
                  [ 5  2  8 | 1  9  3 ]
                  [ 5 | 2  8 ]   [ 1 | 9  3 ]
                  [5][2 8]        [1][9 3]
                  [5][2][8]       [1][9][3]   <- 길이 1까지 분할 (이미 정렬됨)

[병합 단계 — 정렬하며 합친다 (▲ = 두 조각 중 더 작은 것을 뽑음)]
   [2][8]  ->  [2 8]      ([2]와 [8] 비교, 2◀ 먼저, 그다음 8)
   [5][2 8]->  [2 5 8]    (5 vs 2 ->2◀, 5 vs 8 ->5◀, 남은 8)
   [9][3]  ->  [3 9]
   [1][3 9]->  [1 3 9]

[마지막 병합: 두 정렬된 절반을 합친다]
   왼쪽:  2 5 8        오른쪽: 1 3 9
   포인터 i,j 가 각 배열 앞을 가리키며 작은 쪽을 결과로 보냄

   비교 2 vs 1 -> 1◀ |  결과: [1]
   비교 2 vs 3 -> 2◀ |  결과: [1 2]
   비교 5 vs 3 -> 3◀ |  결과: [1 2 3]
   비교 5 vs 9 -> 5◀ |  결과: [1 2 3 5]
   비교 8 vs 9 -> 8◀ |  결과: [1 2 3 5 8]
   왼쪽 소진, 오른쪽 9 남음 -> 결과: [1 2 3 5 8 9]

최종:             [ 1  2  3  5  8  9 ]   완료!
```

### 2) 삽입 정렬 (Insertion Sort) — "손에 든 카드 끼워 넣기"

정렬된 앞부분에, 새 원소를 올바른 자리에 끼워 넣는다. 작은 N이나 거의 정렬된 데이터에 빠르다(O(N²)이지만 상수가 작음).

```
[ |5  2  8  1 ]   '|' 왼쪽은 정렬됨. 처음엔 5 하나만 정렬됨
 정렬구간: 5

key=2 :  5 > 2 이므로 5를 오른쪽으로 밀고 그 앞에 2 삽입
[ 2  5 | 8  1 ]
 정렬구간: 2 5

key=8 :  5 < 8 이므로 그대로
[ 2  5  8 | 1 ]
 정렬구간: 2 5 8

key=1 :  8>1, 5>1, 2>1 모두 밀어내고 맨 앞에 삽입
[ 1  2  5  8 ]   완료!
```

## C++ 예제 코드

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    // ── 1. 기본 정렬 (오름차순) ──────────────────────────
    vector<int> a = {5, 2, 8, 1, 9, 3};
    sort(a.begin(), a.end());            // 1 2 3 5 8 9  (평균 O(N log N))

    // ── 2. 내림차순 ─────────────────────────────────────
    sort(a.begin(), a.end(), greater<int>());  // 9 8 5 3 2 1
    // 또는 람다:  sort(a.begin(), a.end(), [](int x, int y){ return x > y; });

    // ── 3. 다중 기준 정렬 ───────────────────────────────
    // 학생: (점수, 이름). 점수 내림차순, 동점이면 이름 오름차순(사전순)
    struct Student { int score; string name; };
    vector<Student> v = {{90, "Bob"}, {90, "Amy"}, {80, "Zoe"}};

    sort(v.begin(), v.end(), [](const Student& p, const Student& q) {
        if (p.score != q.score) return p.score > q.score; // 점수 높은 순
        return p.name < q.name;                           // 동점이면 이름 사전순
    });
    // 결과: (90,"Amy"), (90,"Bob"), (80,"Zoe")

    // ── 4. 정렬 후 이분 탐색 / 중복 제거 ────────────────
    vector<int> b = {3, 1, 4, 1, 5, 9, 2, 6, 5};
    sort(b.begin(), b.end());            // 1 1 2 3 4 5 5 6 9
    // 중복 제거: unique는 "인접한" 중복만 제거하므로 반드시 정렬 먼저!
    b.erase(unique(b.begin(), b.end()), b.end());  // 1 2 3 4 5 6 9

    // 값 5가 있는지: binary_search 는 정렬된 배열에서만 정확하다
    bool found = binary_search(b.begin(), b.end(), 5);   // true
    // 5 이상이 처음 나오는 위치(인덱스):
    int idx = lower_bound(b.begin(), b.end(), 5) - b.begin();

    // ── 5. 안정 정렬이 필요할 때 ────────────────────────
    // 같은 키일 때 입력 순서를 보존해야 한다면 stable_sort 사용
    // stable_sort(v.begin(), v.end(), cmp);

    cout << "found=" << found << " idx=" << idx << "\n";
    for (int x : b) cout << x << ' ';
    cout << "\n";
    return 0;
}
```

핵심 포인트:
- 비교 함수(cmp)는 "p가 q보다 **앞에** 와야 하면 true"를 반환한다. `return p < q`이면 오름차순.
- 비교 함수에서 같을 때 `<=`나 `>=`를 쓰면 **strict weak ordering**이 깨져 런타임 오류가 날 수 있다. 반드시 `<` 또는 `>`만 사용.

## 시간복잡도

| 알고리즘 | 평균 | 최악 | 공간 | 안정성 |
|----------|------|------|------|--------|
| `std::sort` (인트로소트) | O(N log N) | O(N log N) | O(log N) | 불안정 |
| `std::stable_sort` (병합 기반) | O(N log N) | O(N log N) | O(N) | 안정 |
| 병합 정렬 | O(N log N) | O(N log N) | O(N) | 안정 |
| 퀵 정렬 | O(N log N) | O(N²) | O(log N) | 불안정 |
| 힙 정렬 | O(N log N) | O(N log N) | O(1) | 불안정 |
| 삽입/선택/버블 | O(N²) | O(N²) | O(1) | 삽입·버블만 안정 |
| 카운팅 정렬 | O(N+K) | O(N+K) | O(K) | 안정 |

- **비교 정렬의 이론적 하한은 O(N log N)** 이다. 그보다 빠르려면(카운팅/기수) 값의 성질(범위가 작은 정수 등)을 이용해야 한다.
- 코딩테스트에서 N ≤ 약 100만이면 O(N log N) 정렬이 무난히 통과한다. O(N²)은 N이 1만만 넘어도 위험하다.

## 흔한 실수 / 팁

- **비교 함수에 등호(`<=`, `>=`)를 쓰지 말 것.** `return a <= b;`는 strict weak ordering 위반으로 `std::sort`가 메모리 침범/런타임 에러를 일으킬 수 있다. 항상 `<` 또는 `>`.
- **`binary_search` / `lower_bound` 전에는 반드시 정렬되어 있어야 한다.** 안 그러면 결과가 엉뚱하게 나온다.
- **`unique`는 인접 중복만 제거한다.** 정렬 없이 쓰면 떨어져 있는 중복이 남는다. `sort` → `unique` → `erase` 3단 콤보로 기억하자.
- **안정성이 필요하면 `stable_sort`.** 같은 키의 원래 순서를 보존해야 하는 문제(예: 입력 순서 유지)에서 `sort`는 순서를 보장하지 않는다.
- **정렬 후 인덱스가 바뀐다.** 원래 위치가 중요하면 `(값, 원래인덱스)` 쌍으로 묶어서 정렬하거나, 인덱스 배열을 만들어 간접 정렬하라.
- **오버플로 주의.** 비교 함수에서 `a.x - b.x` 같은 뺄셈으로 대소를 판단하면 int 오버플로가 날 수 있다. `a.x < b.x`처럼 비교 연산자를 직접 쓰자.
- **`greater<int>()` 활용.** 내림차순은 람다 대신 `sort(v.begin(), v.end(), greater<int>())`가 간결하다. (단, 사용자 정의 타입엔 람다/연산자 오버로딩 필요)
- **부분 정렬이 필요하면 `partial_sort` 또는 `nth_element`.** 상위 k개만 필요하면 전체 정렬보다 빠르다(`nth_element`는 평균 O(N)).
- **`pair`/`tuple`은 사전식 비교**가 기본이라, 기준 우선순위대로 멤버를 배치하면 비교 함수 없이도 다중 기준 정렬이 된다.
