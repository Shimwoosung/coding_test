# 백트래킹 (Backtracking)

## 한 줄 요약

가능한 모든 경우를 **트리처럼 한 갈래씩 깊게 탐색**하되, "이 길은 더 가봐야 답이 안 나온다"고 판단되면 **즉시 되돌아가서(가지치기)** 다른 갈래를 시도하는 완전탐색 기법이다.

---

## 언제 쓰나

"모든 경우의 수를 만들어서 그중 조건을 만족하는 것을 찾아라" 류의 문제에 거의 다 쓰인다.

- **순열 / 조합 / 부분집합 만들기**: N개 중 M개 고르기, 1~N 숫자로 순열 만들기
- **N-Queen**: N×N 체스판에 퀸 N개를 서로 공격 못 하게 놓기
- **스도쿠**: 빈 칸에 1~9를 채워 규칙을 만족시키기
- **미로/경로 찾기**: 출발점에서 도착점까지 가는 모든 경로 탐색
- **단어/수식 만들기**: 연산자 끼워넣기, 알파벳 배치
- **분할 문제**: 집합을 합이 같은 두 그룹으로 나누기
- 한마디로 **"선택 → 그 선택을 바탕으로 다음 선택 → 막히면 취소(undo)하고 다른 선택"** 구조면 백트래킹이다.

---

## 왜 이걸 쓰나 + 대안과 차이

### 단순 완전탐색(브루트포스)과의 차이

브루트포스는 **모든 경우를 끝까지 다 만들어 본 뒤** 조건을 검사한다. 반면 백트래킹은 **만드는 도중에 "이미 글렀다"는 걸 알면 그 즉시 멈추고 되돌아간다.** 이 "가지치기(pruning)" 덕분에 탐색해야 할 경우의 수가 폭발적으로 줄어든다.

```
예: 1~4 숫자로 "앞 자리보다 큰 수만" 나열하기

[브루트포스]                    [백트래킹]
1을 골랐다                       1을 골랐다
 -> 1 다음에 1? (안 됨) 끝까지   -> 다음엔 2,3,4만 시도 (1은 시도조차 안 함)
 -> 1 다음에 2? ...             -> 막히는 가지를 아예 안 들어감
 모든 조합 생성 후 검사          유효한 가지만 내려감 = 빠름
```

### 다른 기법과의 비교

| 기법 | 특징 | 백트래킹과의 관계 |
|------|------|------------------|
| **DFS** | 그래프/트리를 깊이 우선으로 탐색 | 백트래킹은 "상태 공간 트리에 대한 DFS + 가지치기 + 상태 되돌리기" |
| **브루트포스** | 전부 만들고 검사 | 백트래킹은 만들다가 중간에 포기 가능 |
| **DP** | 겹치는 부분문제를 저장해 재사용 | 답의 "개수/최적값"이 목적이고 부분문제가 겹치면 DP가 유리. 백트래킹은 "모든 해의 구성 자체"가 필요할 때 |
| **BFS** | 너비 우선, 최단거리에 강함 | 백트래킹은 "조합/배치 생성"에 강함. 메모리는 보통 백트래킹(재귀 스택)이 더 적음 |

핵심: **백트래킹 = DFS 위에 "유망하지 않으면 되돌아간다"는 가지치기를 얹은 것.**

## 동작 과정

예제: **1, 2, 3 중에서 서로 다른 2개를 골라 순열 만들기** (N=3, M=2)

`visited[]`로 이미 쓴 숫자를 표시하고, 다 고르면 출력 후 되돌린다.

```
상태 공간 트리 (깊이 = 고른 개수)

                       (시작: [])
        ┌───────────────┼───────────────┐
       1 선택          2 선택          3 선택
      [1]             [2]             [3]
    ┌──┼──┐         ┌──┼──┐         ┌──┼──┐
   (1막) 2  3      1 (2막) 3       1  2 (3막)
       [1,2][1,3] [2,1]  [2,3]   [3,1][3,2]
        ✔   ✔     ✔      ✔       ✔    ✔

(N막) = 이미 visited 라서 가지치기로 들어가지 않음
✔ = M개 채움 -> 정답 출력 후 되돌아감(undo)
```

단계별로 visited 배열과 current 경로가 어떻게 변하는지 추적:

```
step  동작                     current   visited[1,2,3]   비고
----  -----------------------  --------  ---------------  ----------------------
 1    1 선택 (visited[1]=T)     [1]       T F F
 2     2 선택                   [1,2]     T T F            크기=2 -> 출력 "1 2"
 3     2 되돌림(undo)           [1]       T F F            visited[2]=F
 4     3 선택                   [1,3]     T F T            크기=2 -> 출력 "1 3"
 5     3 되돌림                 [1]       T F F
 6    1 되돌림                  []        F F F            첫 자리 1 가지 끝
 7    2 선택                    [2]       F T F
 8     1 선택                   [2,1]     T T F            출력 "2 1"
 9     1 되돌림                 [2]       F T F
10     3 선택                   [2,3]     F T T            출력 "2 3"
11     3 되돌림                 [2]       F T F
12    2 되돌림                  []        F F F
13    3 선택 ... (이하 동일하게 3 1, 3 2 생성)
```

가장 중요한 패턴: **"선택했으면 반드시 그만큼 되돌린다(undo)".** 재귀를 빠져나온 직후 `visited`를 다시 false로, 경로에서 원소를 다시 빼는 이 한 줄이 백트래킹의 심장이다.

### 가지치기가 보이는 예: 합이 정확히 6인 조합만

```
1~4 중 골라 합=6 만들기. 합이 6을 "넘으면" 더 안 내려감(가지치기)

[1]                                    
 ├ [1,2]                               
 │   ├ [1,2,3] sum=6 ✔ 출력!           
 │   └ [1,2,4] sum=7 > 6  ✘ 가지치기   <- 4 이후 가지 통째로 스킵
 ├ [1,3]                               
 │   └ [1,3,4] sum=8 > 6  ✘ 가지치기   
 └ [1,4]                               
     └ [1,4,...]  sum=5, 더 큰 수 없음   
[2]
 └ [2,4] sum=6 ✔ 출력!
 ...
```

`if (sum > target) return;` 한 줄이 수많은 가지를 통째로 잘라낸다.

## C++ 예제 코드

### 예제 1: N개 중 M개 순열 (백준 N과 M 류의 기본형)

```cpp
#include <bits/stdc++.h>
using namespace std;

int N, M;
int path[20];        // 현재까지 고른 숫자들을 담는 경로
bool visited[20];    // 각 숫자를 이미 썼는지 표시

void backtrack(int depth) {
    // [종료 조건] M개를 다 골랐으면 하나의 해 완성 -> 출력
    if (depth == M) {
        for (int i = 0; i < M; i++)
            cout << path[i] << ' ';
        cout << '\n';
        return;          // 이 갈래는 끝났으니 되돌아감
    }

    // [선택 분기] 1부터 N까지 후보를 하나씩 시도
    for (int num = 1; num <= N; num++) {
        if (visited[num]) continue;   // 가지치기: 이미 쓴 숫자는 건너뜀

        // ----- 선택(choose) -----
        visited[num] = true;
        path[depth] = num;

        backtrack(depth + 1);         // 다음 자리를 채우러 깊이 내려감

        // ----- 되돌리기(undo) ----- 핵심!
        visited[num] = false;         // 표시를 풀어서 다른 갈래에서 재사용 가능
        // path[depth]는 다음 루프에서 덮어쓰므로 굳이 지울 필요는 없다
    }
}

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    cin >> N >> M;        // 예: 3 2
    backtrack(0);         // 0번째 자리부터 채우기 시작
    return 0;
}
```

### 예제 2: N-Queen (가지치기가 진가를 발휘하는 예)

```cpp
#include <bits/stdc++.h>
using namespace std;

int N;
int col[16];      // col[r] = r행에 놓은 퀸의 열 위치
int answer = 0;

// r행에 놓은 퀸이 앞선 0..r-1행의 퀸들과 충돌하지 않는지 검사
bool isSafe(int r) {
    for (int prev = 0; prev < r; prev++) {
        // 같은 열에 있으면 충돌
        if (col[prev] == col[r]) return false;
        // 대각선 충돌: 행 차이 == 열 차이
        if (abs(col[prev] - col[r]) == abs(prev - r)) return false;
    }
    return true;
}

void backtrack(int r) {
    if (r == N) {            // N개 행에 모두 놓는 데 성공
        answer++;
        return;
    }
    for (int c = 0; c < N; c++) {
        col[r] = c;          // r행 퀸을 c열에 놓아본다 (선택)
        if (isSafe(r)) {     // 가지치기: 안전할 때만 다음 행으로 내려감
            backtrack(r + 1);
        }
        // col[r]은 다음 c에서 덮어써지므로 별도 undo 불필요
    }
}

int main() {
    cin >> N;                // 예: 8 -> 92 출력
    backtrack(0);
    cout << answer << '\n';
    return 0;
}
```

포인트: `isSafe`가 false면 그 배치에서 **더 깊이 내려가지 않는다.** 이것이 가지치기다. 만약 가지치기 없이 모든 배치를 다 만든 뒤 검사하면 N=8에서도 엄청 느려지지만, 백트래킹은 즉시 잘라내므로 매우 빠르다.

## 시간복잡도

백트래킹은 본질적으로 완전탐색이라 **최악의 경우는 지수/팩토리얼급**이다. 다만 가지치기가 실제로 얼마나 잘리느냐에 따라 실제 수행 시간은 크게 달라진다.

| 문제 유형 | 최악 시간복잡도 |
|-----------|-----------------|
| N개 중 M개 순열 | O(N! / (N-M)!) |
| N개 부분집합 (각 원소 포함/제외) | O(2^N) |
| 순열 전체 | O(N! · N) (출력 포함) |
| N-Queen | 상한 O(N!) 이지만 가지치기로 훨씬 빠름 |

- **가지치기가 잘 들어가면 이론 상한보다 훨씬 빠르게 끝난다.** 그래서 "유망성 검사(promising check)"를 얼마나 빨리, 강하게 거느냐가 핵심 최적화 포인트다.
- 입력 N이 작을 때(보통 N ≤ 10~20 수준) 백트래킹이 적합하다. N이 크면 DP나 그리디 등 다른 접근을 고려해야 한다.

---

## 흔한 실수 / 팁

- **되돌리기(undo)를 빼먹는 실수가 1순위.** 선택할 때 바꾼 상태(`visited`, 경로, 합, 보드 등)는 재귀에서 돌아온 뒤 **정확히 원래대로** 복구해야 한다. 안 그러면 다른 갈래가 오염되어 답이 틀린다.
  ```cpp
  visited[num] = true;
  backtrack(depth + 1);
  visited[num] = false;   // <- 이걸 빼먹으면 끝장
  ```
- **종료 조건을 먼저, 정확히.** `if (depth == M)`처럼 "해가 완성되는 시점"을 함수 맨 위에서 처리하고 `return`하라. 빠뜨리면 무한 재귀나 인덱스 초과가 난다.
- **가지치기는 가능한 한 일찍.** 합이 이미 목표를 넘었으면 더 내려가지 말고 즉시 `return`. 깊이 다 내려간 뒤 검사하면 백트래킹의 의미가 없다.
- **조합 vs 순열 구분.** 순서가 의미 없으면(조합) 다음 재귀에 `start` 인덱스를 넘겨 `for (i = start; ...)`로 중복/순서 뒤집힘을 막는다. 순서가 의미 있으면(순열) `visited`로 사용 여부만 관리한다.
- **전역 vs 매개변수.** 경로 배열이나 `visited`는 전역으로 두면 매번 복사 비용이 없어 빠르다. 단, undo를 반드시 책임져야 한다.
- **출력이 느릴 때.** 해가 매우 많으면 `cout`보다 `'\n'` 사용, `sync_with_stdio(false)`, 또는 결과를 모아 한 번에 출력하는 것이 유리하다.
- **재귀 깊이 = 트리 깊이.** 보통 N이 작아서 스택 오버플로 걱정은 적지만, 깊이가 깊어질 수 있는 문제라면 주의하라.
