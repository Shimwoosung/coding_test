# 백트래킹 (Backtracking)

## 한 줄 요약
가능한 경우를 재귀로 만들어가다, 조건을 어기면 되돌아가(가지치기) 탐색 공간을 줄인다.

## 언제 쓰나
- 순열/조합 생성(N과 M), N-Queen, 스도쿠
- 연산자 끼워넣기, 조건부 문자열 생성(암호 만들기)

## 왜 이거 + 대안 + 차이
- 완전탐색이지만 "안 될 게 뻔한 가지"를 일찍 잘라 실제 탐색량을 크게 줄인다.
- DP는 부분 문제 재사용, 백트래킹은 해 자체를 구성. 해의 "나열"이 필요하면 백트래킹.

## 핵심 틀
- 선택 → 재귀 → 선택 취소(원상복구). used 배열 또는 start 인덱스.
- 순열=used 배열, 조합=start 인덱스로 증가만 허용.
- 가지치기 조건을 재귀 진입 전에 검사할수록 빠름.

## 흔한 실수
- 선택 취소(pop_back, used=false) 누락 → 상태 오염
- 조합인데 순열처럼 짜서 중복 생성
