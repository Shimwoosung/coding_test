# 분리 집합 (Union-Find / DSU)

## 한 줄 요약
"이 둘이 같은 그룹인가?"와 "두 그룹 합치기"를 거의 O(1)에. 연결성·MST의 핵심.

## 언제 쓰나
- 같은 집합 여부 질의, 연결 요소, 사이클 탐지
- 최소 스패닝 트리(크루스칼), 도시 분할

## 왜 이거 + 대안 + 차이
- 매번 BFS로 연결성 확인하면 느림. DSU는 find/union이 사실상 상수.
- 경로 압축 + 랭크(사이즈) 합치기로 거의 O(α(N)).

## 핵심
- `int find(x){ return p[x]==x?x:p[x]=find(p[x]); }` (경로 압축)
- union: `p[find(a)] = find(b);`
- 크루스칼: 간선 가중치 정렬 → 사이클 안 생기면 union.

## 흔한 실수
- find 없이 부모 배열 직접 비교
- 초기화(p[i]=i) 누락
