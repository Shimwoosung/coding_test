# 해시 (Hash) & 해시맵 / 해시셋

## 한 줄 요약
해시는 **"키(key)를 정수 인덱스로 바꾸는 함수"**를 이용해, 값의 저장과 조회를 평균 O(1)에 처리하는 기법이다. 코딩테스트에서 "존재하는가?", "몇 번 나왔는가?", "짝을 찾아라" 같은 문제를 빠르게 푸는 핵심 도구다.

## 언제 쓰나
- **중복 검사**: 배열에 같은 값이 두 번 이상 나오는지 확인 (`unordered_set`)
- **빈도수 세기**: 각 원소가 몇 번 등장했는지 카운트 (`unordered_map<값, 개수>`)
- **존재 여부 빠른 확인**: "이 단어가 사전에 있나?", "이 좌표를 방문했나?"
- **두 수의 합 (Two Sum)**: `target - 현재값`이 이미 있었는지 즉시 확인
- **그룹핑 / 분류**: 애너그램 묶기, 카테고리별 모으기
- **캐시 / 메모이제이션**: 이미 계산한 입력의 결과를 키로 저장
- **매핑 관계 저장**: "학번 -> 이름", "문자 -> 빈도" 같은 1:1 또는 1:N 대응

## 왜 이걸 쓰나 + 대안과 차이

| 자료구조 | 탐색 | 삽입/삭제 | 정렬 순서 | 비고 |
|---|---|---|---|---|
| 배열 선형 탐색 | O(N) | O(1)~O(N) | 입력 순 | 느리지만 단순 |
| 정렬 배열 + 이분 탐색 | O(log N) | O(N) | 정렬됨 | 삽입 비쌈 |
| `std::map` (균형 BST) | O(log N) | O(log N) | **키 정렬됨** | 순서가 필요할 때 |
| **`std::unordered_map` (해시)** | **평균 O(1)** | **평균 O(1)** | **없음** | 가장 빠름, 순서 X |

**핵심 차이**
- 배열을 매번 처음부터 훑으면 O(N). N개를 N번 검사하면 O(N^2)으로 터진다. 해시는 한 번 조회를 O(1)로 줄여 전체를 O(N)으로 만든다.
- `map`은 내부적으로 트리라서 **키가 자동 정렬**되고 안정적인 O(log N)을 보장한다. "정렬된 순서로 순회"나 "특정 값 이상 범위 조회"가 필요하면 `map`.
- `unordered_map`은 평균 O(1)로 더 빠르지만 **순서가 보장되지 않고**, 최악의 경우(해시 충돌이 몰릴 때) O(N)까지 느려질 수 있다. 순서가 필요 없고 속도가 중요하면 `unordered_map`.

## 동작 과정

해시 테이블의 핵심은 **해시 함수**로 키를 버킷(bucket, 배열의 칸) 번호로 바꾸는 것이다.
예시로 `해시값 = key % 7` (버킷 7개)을 사용해 `{15, 22, 8, 29}`를 넣어보자.

```
[1단계] 빈 해시 테이블 (버킷 7개)
 index:   0    1    2    3    4    5    6
        [ - ][ - ][ - ][ - ][ - ][ - ][ - ]

[2단계] 15 삽입 ->  15 % 7 = 1  ->  버킷 1
 index:   0    1    2    3    4    5    6
        [ - ][15 ][ - ][ - ][ - ][ - ][ - ]
                ^^
              여기!

[3단계] 22 삽입 ->  22 % 7 = 1  ->  버킷 1  (충돌! 이미 15가 있음)
        체이닝(chaining): 같은 버킷에 연결 리스트로 이어 붙인다
 index:   0    1    2    3    4    5    6
        [ - ][15 ][ - ][ - ][ - ][ - ][ - ]
                |
                v
              [22 ]      <- 충돌난 값은 뒤에 매단다

[4단계] 8 삽입 ->  8 % 7 = 1  ->  버킷 1  (또 충돌!)
 index:   0    1    2    3    4    5    6
        [ - ][15 ][ - ][ - ][ - ][ - ][ - ]
                |
                v
              [22 ] -> [ 8 ]

[5단계] 29 삽입 ->  29 % 7 = 1  ->  버킷 1  (역시 충돌!)
 index:   0    1    2    3    4    5    6
        [ - ][15 ][ - ][ - ][ - ][ - ][ - ]
                |
                v
              [22 ] -> [ 8 ] -> [29 ]
```

위 예시는 일부러 `% 7`로 충돌을 만들어 보여준 것이다. 실제 라이브러리는 좋은 해시 함수와 충분한 버킷 수, 자동 리해싱(rehashing)으로 충돌을 줄여 평균 O(1)을 유지한다.

**조회 과정** — "8이 있나?" 검색:

```
1) 해시 계산:  8 % 7 = 1        -> 버킷 1만 보면 됨 (전체를 훑지 않음!)
2) 버킷 1의 리스트를 따라가며 비교:
      15  ? 8 -> 아니오
      22  ? 8 -> 아니오
       8  ? 8 -> 예! 찾음 ✓
```

전체 배열을 훑었다면 4칸을 봐야 했지만, 해시는 버킷 1만 확인한다. 충돌이 적으면 사실상 1~2번 비교로 끝난다.

### 빈도수 세기 시각화 (`unordered_map<int,int>`)

입력: `[2, 7, 2, 5, 7, 2]` 를 차례로 카운트

```
처리 전:   {}
2 처리 ->  {2:1}
7 처리 ->  {2:1, 7:1}
2 처리 ->  {2:2, 7:1}        (2의 카운트 +1)
5 처리 ->  {2:2, 7:1, 5:1}
7 처리 ->  {2:2, 7:2, 5:1}
2 처리 ->  {2:3, 7:2, 5:1}   (최종)
```
각 원소를 볼 때 `cnt[x]++` 한 번이면 끝 — 한 번 훑어 O(N).

## C++ 예제 코드

```cpp
#include <iostream>
#include <unordered_map>
#include <unordered_set>
#include <vector>
using namespace std;

int main() {
    // ───────────────────────────────────────────────
    // 1) unordered_set : "있는지 없는지" / 중복 검사
    // ───────────────────────────────────────────────
    unordered_set<int> visited;
    visited.insert(10);
    visited.insert(20);

    // count()는 0 또는 1을 반환 (set은 중복 저장 안 함)
    if (visited.count(10))                 // 평균 O(1)
        cout << "10은 이미 방문함\n";
    if (visited.find(30) == visited.end()) // find: 없으면 end() 반환
        cout << "30은 아직 안 봄\n";

    // 중복 검사 예: 배열에 같은 값이 두 번 있는가?
    vector<int> arr = {3, 5, 7, 5, 9};
    unordered_set<int> seen;
    bool hasDup = false;
    for (int x : arr) {
        if (seen.count(x)) { hasDup = true; break; } // 이미 봤다면 중복!
        seen.insert(x);
    }
    cout << "중복 존재? " << (hasDup ? "예" : "아니오") << "\n"; // 예 (5)

    // ───────────────────────────────────────────────
    // 2) unordered_map : "키 -> 값" 매핑 / 빈도수 세기
    // ───────────────────────────────────────────────
    vector<int> nums = {2, 7, 2, 5, 7, 2};
    unordered_map<int, int> freq;
    for (int x : nums)
        freq[x]++;   // 키가 없으면 자동으로 0에서 시작해 +1 (핵심 관용구!)

    for (auto& [key, cnt] : freq)          // 구조적 바인딩 (C++17)
        cout << key << "는 " << cnt << "번 등장\n";
    // 주의: 출력 순서는 보장되지 않는다 (정렬 X)

    // ───────────────────────────────────────────────
    // 3) Two Sum : 합이 target이 되는 두 인덱스 찾기 (O(N))
    // ───────────────────────────────────────────────
    vector<int> a = {2, 11, 7, 15};
    int target = 9;
    unordered_map<int, int> pos;           // {값 -> 인덱스}
    for (int i = 0; i < (int)a.size(); i++) {
        int need = target - a[i];          // 짝이 되어야 할 값
        if (pos.count(need)) {             // 그 값을 이미 본 적 있다면 정답!
            cout << "정답 인덱스: " << pos[need] << ", " << i << "\n"; // 0, 2
            break;
        }
        pos[a[i]] = i;                     // 현재 값을 기록해 둔다
    }

    // ───────────────────────────────────────────────
    // 4) 문자열 키도 가능: 단어 빈도 등
    // ───────────────────────────────────────────────
    unordered_map<string, int> wordCount;
    wordCount["apple"]++;
    wordCount["apple"]++;
    cout << "apple: " << wordCount["apple"] << "\n"; // 2

    return 0;
}
```

## 시간복잡도

| 연산 | 평균 | 최악 |
|---|---|---|
| 삽입 `insert` / `[]` | O(1) | O(N) |
| 조회 `find` / `count` | O(1) | O(N) |
| 삭제 `erase` | O(1) | O(N) |
| 전체 순회 | O(N) | O(N) |

- **평균 O(1)**: 좋은 해시 함수로 충돌이 거의 없을 때.
- **최악 O(N)**: 모든 키가 같은 버킷으로 몰릴 때(악의적 입력 또는 나쁜 해시). 대부분의 코테에서는 평균을 가정해도 안전하다.
- 비교 대상: `std::map`은 항상 O(log N) **보장**. 안정적인 보장이 필요하면 `map`.

## 흔한 실수 / 팁

- **`[]` 접근은 키를 만든다**: `m[key]`로 읽기만 했는데 키가 없으면 **기본값(0)으로 새로 삽입**된다. 단순 존재 확인은 `m.count(key)`나 `m.find(key)`를 써라.
  ```cpp
  if (m[key] > 0) ...   // key가 없었다면 m[key]=0이 새로 생겨버림 (의도치 않은 삽입)
  if (m.count(key) && m[key] > 0) ...  // 안전
  ```
- **순서에 의존하지 말 것**: `unordered_map`/`unordered_set`은 순회 순서가 **삽입 순도 정렬 순도 아니다**. 출력에 순서가 필요하면 `map`을 쓰거나, 키를 따로 벡터에 모아 정렬하라.
- **`pair`나 사용자 정의 타입을 키로** 쓰려면 해시 함수가 없어 컴파일 에러가 난다. 직접 해시 함수를 정의하거나, 간단히 `map<pair<int,int>, T>`(트리 기반은 `<` 연산만 있으면 됨)를 쓰는 것이 편하다.
- **성능 안정성이 필요하면 reserve**: 원소 개수를 안다면 `m.reserve(N)`으로 리해싱을 줄여 빨라진다.
- **빈도수 관용구 `cnt[x]++`** 를 외워라. 키가 없으면 0에서 시작하므로 초기화 없이 바로 증가시킬 수 있다.
- **중복을 허용하려면** `unordered_multiset` / `unordered_multimap`. 일반 `set`/`map`은 같은 키를 하나만 유지한다.
- **방문 체크에 좌표를 쓸 때**: 2차원 좌표는 `set<pair<int,int>>`보다, 격자가 작다면 `bool visited[H][W]` 2차원 배열이 더 빠르고 단순하다. 해시가 항상 정답은 아니다.
