# 이분 탐색 (Binary Search)

## 한 줄 요약
정렬된(또는 단조성이 있는) 범위에서 정답 후보를 매번 절반으로 줄여 **O(log N)** 에 찾는 방법.

## 언제 쓰나
- 정렬된 배열에서 값 찾기
- "조건을 만족하는 **최소/최대값**" 을 찾는 매개변수 탐색(Parametric Search)
- 범위가 큰데(예: 1e9) 선형 탐색이면 시간초과 날 때

## 왜 이거 + 대안 + 차이
- 선형 탐색 **O(N)** vs 이분 탐색 **O(log N)**. N이 1e9면 선형은 10억 번, 이분은 약 30번.
- 단, 이분 탐색은 **단조성(monotonic)** 이 있어야 한다 — 한 방향으로 정렬/증가해야 절반을 버릴 수 있다.
- 해시(unordered_map)는 "특정 값의 존재"는 O(1)이지만, "조건을 만족하는 경계값"은 못 찾는다. 경계 탐색이 필요하면 이분 탐색.

## 모양만 기억 (암기 X)

```
lo, hi = 범위
while (lo < hi) {
    mid = (lo + hi) / 2;       // 최대값 찾을 땐 (lo+hi+1)/2
    if (조건 만족) lo = mid;    // 또는 hi = mid
    else hi = mid - 1;         // 또는 lo = mid + 1
}
정답 = lo
```

## 그림으로 보기

```
찾는 값 = 7,  배열 = [1 3 5 7 9 11 13]
                      ^lo        ^hi
 mid ──────────────────────┘ (=7?) → 작으면 오른쪽, 크면 왼쪽 버림
 매 단계 후보 구간이 절반으로 줄어든다:
 [1 3 5 7 9 11 13] → [7 9 11 13] → [7 9] → [7] ✔
```

## 짧은 예제 (lower_bound 직접 구현)

```cpp
// v 에서 x 이상이 처음 나오는 위치
int lo = 0, hi = v.size();
while (lo < hi) {
    int mid = lo + (hi - lo) / 2;
    if (v[mid] >= x) hi = mid;
    else lo = mid + 1;
}
// lo 가 답. STL 로는 lower_bound(v.begin(), v.end(), x) - v.begin();
```

## 흔한 실수
- mid 계산 시 오버플로 → `(lo+hi)/2` 대신 `lo + (hi-lo)/2`.
- 무한 루프 → 최대값 탐색인데 mid를 내림으로 잡으면 lo가 안 올라감. `+1` 필요.
- 경계(lo==hi) 처리 / 닫힌구간 vs 반열린구간 일관성 유지.
