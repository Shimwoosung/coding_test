# 이분 탐색 (Binary Search)

## 한 줄 요약
정렬되었거나 단조성이 있는 범위에서 정답 후보를 매번 절반으로 줄여 O(log N)에 찾는다.

## 언제 쓰나
- 정렬된 배열에서 값/개수 찾기 (lower_bound, upper_bound)
- "조건을 만족하는 최소/최대값"을 찾는 매개변수 탐색(Parametric Search) — 랜선 자르기, 나무 자르기, 예산
- 범위가 큰데(1e9 등) 선형 탐색이면 시간초과

## 왜 이거 + 대안 + 차이
- 선형 탐색 O(N) vs 이분 탐색 O(log N). N=1e9면 10억 번 vs 약 30번.
- 전제: "단조성" — 한 방향으로 정렬/증가해야 절반을 버릴 수 있다.

## 모양만 기억 (암기 X)
- 최소값 찾기: mid=(lo+hi)/2, 조건 만족 시 hi=mid, 아니면 lo=mid+1
- 최대값 찾기: mid=lo+(hi-lo+1)/2, 조건 만족 시 lo=mid, 아니면 hi=mid-1

## 흔한 실수
- 오버플로 → (lo+hi)/2 대신 lo+(hi-lo)/2
- 최대값 탐색인데 mid를 내림으로 잡으면 무한 루프 → +1 필요
- mid*mid 같은 곱은 long long
